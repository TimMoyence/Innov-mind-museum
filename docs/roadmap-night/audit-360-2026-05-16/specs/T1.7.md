## Design

### Helpers à extraire / Zod schemas à créer

**Site 1 (semaphore)** — Co-localisé dans `semaphore.ts`. Pas de helper extrait (1 ligne).
- Replace : `return undefined as unknown as T;` → `return undefined as never;`
- TS accepte `never` assignable à `T`, sémantiquement correct (cette ligne n'est jamais lue car timeout reject win).
- Alternative si TS strict refuse : helper `unreachableTimeout<T>(): T { throw new Error('unreachable: timeout path'); }` — mais change runtime (jette au lieu de retourner) → risque de double rejection. **Recommandation : `as never` simple.**

**Site 2 (guardrail metadata)** — Co-localisé fonction inline ou top-of-file.
```ts
const metadataToRecord = (m: ChatAssistantMetadata): Record<string, unknown> => ({ ...m });
```
Spread shallow → TS l'accepte comme `Record<string, unknown>`. Aucune lossy semantique (sites consommateurs lisent par `Object.entries` / index access).

**Site 3 (enrichment-cache jsonb)** — Co-localisé, remplace `parsedToJsonb` :
```ts
function parsedToJsonb(value: ParsedOpeningHours | null): Record<string, unknown> | null {
  return value === null ? null : { ...value };
}
```
Idem pattern spread. Le commentaire doc actuel "Single cast site for the variance gap" devient obsolète — réécrire la doc.

**Site 4 (data-source driver)** — Type predicate co-localisé :
```ts
interface PgPoolDriver {
  master: { totalCount: number; idleCount: number; waitingCount: number };
}
function hasPgPool(d: unknown): d is PgPoolDriver {
  return (
    typeof d === 'object' && d !== null && 'master' in d &&
    typeof (d as { master: unknown }).master === 'object' &&
    (d as { master: unknown }).master !== null
  );
}
// Usage:
if (!hasPgPool(AppDataSource.driver)) return;
const { totalCount, idleCount, waitingCount } = AppDataSource.driver.master;
```

**Site 5 (social-otc-store)** — Schema Zod en ctor param :
```ts
export class RedisSocialOtcStore<TPayload> implements SocialOtcStore<TPayload> {
  constructor(redis: Redis, options: { ttlSeconds?: number; schema: ZodType<TPayload> }) { /* ... */ }
  async consume(code: string): Promise<TPayload | null> {
    /* ... */
    const parsed = this.schema.safeParse(JSON.parse(value));
    return parsed.success ? parsed.data : await this.fallback.consume(code);
  }
}
```
Idem `InMemorySocialOtcStore` (cohérence). `createSocialOtcStore<TPayload>(redis?, schema)` builder mis à jour. Call-sites (auth composition root) doivent passer `socialOtcPayloadSchema`.
