## Design

### Recommandation : **path (a) cull** des 5 sous-exports phantom

Justification (UFR-016 strict) :
- 5/5 sous-exports = 0 hit path-style + 0 hit named-export root-barrel ⇒ doctrine `bury_dead_code` s'applique sans nuance.
- Les apps ont déjà des implémentations locales (BE haversine, FE haversine ; BE jwt-decode, FE jwt-decode ; BE locale, FE supportedLocales) — pas de besoin produit identifié pour les remonter dans `@musaium/shared` à 16 jours du launch.
- Path (b) coûte 4h pour câbler 5 imports artificiels (UFR-005 « no piling ») sans valeur immédiate. À reporter post-V1 si DRY cross-app devient un sujet.
- Le cull réduit la surface du `file:` package, simplifie le `pnpm bootstrap`, et clarifie la promesse du paquet (« sentry-scrubber centralisé »).

### Path (a) cull — détails opérationnels

**Fichiers à supprimer (entièrement) :**
- `packages/musaium-shared/src/geo/` (2 fichiers : `index.ts`, `haversine.ts`)
- `packages/musaium-shared/src/auth/` (2 fichiers : `index.ts`, `jwt-decode.ts`)
- `packages/musaium-shared/src/errors/` (2 fichiers : `index.ts`, `codes.ts`)
- `packages/musaium-shared/src/i18n/` (2 fichiers : `index.ts`, `locales.ts`)
- `packages/musaium-shared/src/validation/` (2 fichiers : `index.ts`, `password.ts`)

**Modifications fichiers :**
- `packages/musaium-shared/src/index.ts` : retirer les 5 lignes `export * from './geo'`, `'./validation'`, `'./i18n'`, `'./errors'`, `'./auth'`. Garder uniquement `export * from './observability';` (le BE consumer importe `scrubEvent`/`shouldDropBreadcrumb`/`REDACTED`/`SENSITIVE_QUERY_KEYS`/`ScrubbableBreadcrumb`/`ScrubbableEvent` via root barrel — cette ligne préserve cette compatibilité).
- `packages/musaium-shared/package.json` :
  - Retirer les 5 entrées `"./geo"`, `"./validation"`, `"./i18n"`, `"./errors"`, `"./auth"` du champ `exports` (lignes 13-32). Garder `"."` et `"./observability"`.
  - Bump version (semver — voir Open Question Q2). Recommandation : **`0.2.0`** (bump MINOR, package marqué `private: true` + version `0.1.0` ⇒ pas de SemVer strict externe, mais le MINOR signale la rupture API publique du package interne).
  - Mettre à jour le `description` ligne 5 pour retirer mentions `geo, validation, i18n, errors, auth` ; nouveau libellé suggéré : `"Cross-app sentry scrubber + observability primitives — consumed by museum-backend, museum-web, and museum-frontend."`.

**Pipeline de validation :**
1. `cd packages/musaium-shared && pnpm build` → tsc compile clean (dist/ ne contient plus que index.* + observability/*)
2. `cd packages/musaium-shared && pnpm test` → les tests existants `observability/sentry-scrubber.test.ts` passent (déjà couverts par le script).
3. `pnpm bootstrap` (root) → ré-install les 3 apps en séquence (cf. gotcha CLAUDE.md « `@musaium/shared` est un `file:` package, pas un workspace pnpm »).
4. `node scripts/sentinels/workspace-links.mjs` → exit 0 (aucun symlink cassé).
5. Smoke build 3 apps :
   - BE : `cd museum-backend && pnpm lint && pnpm build` (lint inclut `tsc --noEmit`, build inclut `tsc + tsc-alias`).
   - FE : `cd museum-frontend && npm run lint` (tsc --noEmit ; pas de `build` standalone Expo).
   - Web : `cd museum-web && pnpm lint && pnpm build` (lint inclut ESLint + tsc, build Next.js full).
6. Aucun import résiduel à killer dans les 3 apps (vérifié verbatim ci-dessus : 0 hit path-style sur les 5 sous-exports, 0 hit named-export).

### Path (b) câbler — abandonné (pour mémoire)
Demanderait : créer 1 import par sous-export (ex. : BE `nearby-museums.provider.ts` switche `@shared/utils/haversine` → `@musaium/shared/geo` ; BE `social-token-verifier.ts` switche `@shared/auth/jwt-decode` → `@musaium/shared/auth` ; etc.). Effort ~4h. Coût/bénéfice défavorable à J-16. À reconsidérer si une 4ème app rejoint le repo.

## Open Questions

### Q1 — Path (a) cull vs path (b) câbler — confirmation finale ?
**Recommandation spec** : path (a) cull. Justifié par 0 hit confirmé sur 5/5 sous-exports + doctrine UFR-016 + pré-launch J-16 (effort 30min vs 4h, valeur produit nulle pour câbler). À confirmer par owner avant exécution si la roadmap V1.x prévoit déjà une convergence DRY cross-app qui réutiliserait ces helpers (auquel cas path (b) deviendrait pertinent).

### Q2 — Bump semver level
Le package est `"private": true` + version `0.1.0`. SemVer strict ne s'applique pas (pas de release npm). Trois options :
- **0.2.0 (MINOR)** — recommandé. Signale rupture API publique du package interne sans franchir 1.0. Cohérent avec « cull pré-launch d'exports jamais consommés ».
- **1.0.0 (MAJOR)** — défendable pour acter « API stabilisée à observability-only ». Mais 1.0.0 prématuré si le package évoluera post-V1.
- **0.1.1 (PATCH)** — incorrect. La suppression de chemins d'export est un breaking change même pour un package interne (`@musaium/shared/geo` deviendrait `Cannot find module`).

### Q3 — Tests à modifier ?
Aucun. Vérifié verbatim : zéro `*.test.ts` dans `packages/musaium-shared/src/{geo,auth,errors,i18n,validation}/` (seul `observability/sentry-scrubber.test.ts` existe, intact). Les 3 apps n'ont aucun import de ces symbols depuis `@musaium/shared` ⇒ pas de test consumer à updater. Le script `pnpm test` du package (ligne 40 du `package.json` : `node --experimental-strip-types --no-warnings --test src/observability/*.test.ts`) cible déjà uniquement `observability/` ⇒ pas de modification scripts requise.
