## Design

### Étapes bump

1. **Edit `museum-frontend/package.json` lines 40-41** : `^5.99.2` → `^5.100.10` (ou latest 5.x non-major vérifié au green) pour LES DEUX packages (`react-query` + `react-query-persist-client`) en synchro.
2. **`cd museum-frontend && npm install`** (FE = npm, pas pnpm — confirmé par `package-lock.json` présent). Cela régénère `package-lock.json`.
3. **`npm run lint`** (tsc --noEmit) PASS — révèle toute API removed/renamed.
4. **`npm test`** (Node.js test runner via `.test-dist/`) PASS.
5. **Smoke manuel (par user)** : `npm run dev` → chat session list charge OK, museum directory paginé OK, settings TTS voice save OK.
6. **Vérification non-régression `features/chat/useCompareImage.ts`** : `git diff museum-frontend/features/chat/` doit être vide.

### Risques

- **Faible** vu minor bump same-major.
- **Risque résiduel** : peer-deps de `@tanstack/react-query-persist-client` si non bumpé en parallèle (mitigation : bump les deux ensemble).
- **Risque conflict chat-ux** : lockfile uniquement, résolvable au merge.
