# Sub-Processors Inventory — GDPR Art 28 + 44-49 Evidence

**Last review**: 2026-04-26
**Owner**: Tech lead (solo maintainer @TimKraken) + Legal counsel
**Status**: Active — product decisions captured below; DPA URLs flagged `TBD` are accepted residual risk pending legal verification.

### Product decisions (2026-04-26)

- **Sentry**: confirmed EU ingestion (`*.eu.sentry.io`). DSN must use EU region in production.
- **DeepSeek (#2)**: kept as alternative LLM provider. China host, no EU adequacy. Accepted on the basis that LLM provider selection is per-deployment and EU residents are routed to OpenAI/Google by default. SCC + TIA required if/when `LLM_PROVIDER=deepseek` enabled in EU production.
- **Tavily (#15)**: kept as web-search provider. No public DPA. Risk accepted; query payload limited to chat-derived text (no PII). SCC requested in writing from Tavily; outcome tracked in `team-reports/`.
- **Apple / Brave / Unsplash DPAs**: public URLs exist but require legal sign-off — non-blocking.
**Source of truth for env vars**: `museum-backend/src/config/env.ts`

This dossier inventories every third-party data processor reachable from production code in Musaium (museum-backend). Each entry binds the vendor to:

- the env var(s) that gate the integration,
- the endpoint actually called (verified in code, not assumed),
- the data classes transmitted on the wire,
- the GDPR Art 6 lawful basis,
- server location + Art 44-49 transfer mechanism,
- the data-processing agreement (DPA) URL when publicly available.

> Cross-references the audit `team-reports/2026-04-26-security-compliance-full-audit.md` § 2 (Data Categories & Flow Map) and § 4 (V7 — DPA gap → P0).

## 2. Action items (legal verification & remediation)

The following entries require legal review and/or remediation **before** the privacy policy can claim Art 28 compliance. Aligned with audit `V7` (P0 — 24-48h legal review; 7d publication SLA) and `V12` (P1 — breach playbook).

### P0 — block production EU rollout

1. **Sentry DSN region** — confirm production uses `*.eu.sentry.io` (EU-hosted). If US ingestion is in use, switch DSN immediately or document SCC + TIA.
2. **DeepSeek (#2)** — China host, no EU adequacy. Either:
   - **Recommended**: keep `LLM_PROVIDER=openai` or `=google` for EU production; gate DeepSeek behind a non-EU residency flag.
   - Or: execute SCC + TIA + supplementary measures + obtain explicit consent for non-EU transfer (Art 49 derogation flagged for limited use).
3. **S3 provider (#8)** — replace placeholder `https://s3.example.com` in `.env.production.example` with the actual production endpoint and document the vendor + region in this file. Confirm AWS eu-west-3 (Paris) per `docs/AI_VOICE.md` or alternative.
4. **Tavily (#15)** — obtain DPA in writing from Tavily Inc. before EU launch. If no DPA achievable, disable by leaving `TAVILY_API_KEY` empty.

### P1 — within 7 days (alongside audit V7)

5. **DPA inventory in privacy policy** — publish the table above (vendor + purpose + region + DPA URL) under a "Sub-processors" section in `museum-web/.../privacy/` and `docs/privacy-policy.html`.
6. **OpenAI EU data zone** — assess enabling OpenAI's EU residency option for `OPENAI_API_KEY`-using calls (LLM + STT + TTS) to reduce US transfer scope.
7. **Apple Developer DPA (#4)** — locate the public version of the Apple Developer Data Processing Agreement and replace `TBD` here.
8. **Brave Search DPA URL (#16)** — confirm public availability of https://api-dashboard.search.brave.com/legal/data-processing-agreement; replace `TBD`.
9. **Unsplash DPA (#14)** — confirm whether https://unsplash.com/data-processing-addendum is publicly served; otherwise raise with Unsplash support.

### P2 — within 30 days

10. **OTLP collector backend (#7)** — if OTel is enabled in prod, identify the collector vendor (Grafana Cloud, Honeycomb, Datadog, etc.) and add a row to this inventory with full DPA reference.
11. **DuckDuckGo (#17)** — verify whether the client is actually wired into the production `webSearch` chain. If yes, document; if no, delete the adapter (audit dead-code policy).
12. **SearXNG instances (#18)** — for every value in `SEARXNG_INSTANCES`, document operator + DPA in this file before activation.

### Cross-cutting GDPR work (also blocking — see audit § 2)

- ROPA (Art 30) — produce an exhaustive Records of Processing Activities, then keep this file as its sub-processor annex.
- DPIA (Art 35) — the combination of geolocation + AI inference + voice biometrics triggers a DPIA. This sub-processor list is a required input.
- Breach playbook (Art 33/34) — Sentry alert → 72h notification SLA → CNIL portal.

## 3. Verification protocol

Whenever a new env var of shape `*_API_KEY`, `*_DSN`, `*_ENDPOINT`, or any outbound HTTP client is added, a corresponding row MUST be added to this file in the same PR. CI gate proposal (out of scope of this dossier): pre-merge check that `grep -nE 'apiKey|ApiKey|API_KEY|dsn|endpoint' museum-backend/src/config/env.ts` is a strict subset of vendors documented here.
