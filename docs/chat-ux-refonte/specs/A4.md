## 0. Contexte vérifié

### 0.1 État courant — `<ChatHeader>`

Le composant header chat vit à `museum-frontend/features/chat/ui/ChatHeader.tsx` (175 lignes) et est rendu inline dans `museum-frontend/app/(stack)/chat/[sessionId].tsx:384-397`. Findings.md ligne 183 : « P19 — Top bar non-collapsible au scroll — absent — Chrome statique ». Le header occupe ~80dp full-height (titre 2xl + AI badge + 1 à 3 boutons actions ronds 36×36dp `space['9']` + `expo-blur` GlassCard intensity=58) et NE bouge PAS quand l'utilisateur scroll la liste de messages.

Anatomie actuelle :
- **Title row** (`<Text>` `fontSize['2xl']` `fontWeight: '700'` + AI-badge pill EU AI Act Art.50).
- **Sub-row** : `<ExpertiseBadge>` (beginner/intermediate/expert) si défini.
- **Actions row droite** : 1-3 `<Pressable>` ronds (audio-mode toggle, summary, close).
- **Container** : `<GlassCard intensity={58}>` (expo-blur) — déjà translucide cohérent avec A3 assistant bubbles.

### 0.2 Branchement Expo Router header

`museum-frontend/app/(stack)/chat/[sessionId].tsx` est une route Expo Router. Vérification grep : **AUCUN `<Stack.Screen options={…}>` ne configure le header au niveau de cette route** (`grep -n "Stack.Screen\|headerShown" museum-frontend/app/\(stack\)/chat/\[sessionId\].tsx` → 0 résultats). Le screen utilise un layout custom `<ScreenContainer>` qui désactive le header natif Expo Router au niveau du `(stack)` group ; le `<ChatHeader>` est rendu **inline** dans le screen, sous `<KeyboardAvoidingView>` (`[sessionId].tsx:384`).

**Conséquence** : A4 ne touche pas la stack navigator Expo Router. La transition collapsed↔expanded se fait par animation de la hauteur du `<ChatHeader>` rendu inline. ZÉRO migration de header vers `Stack.Screen options.header`. ZÉRO interaction avec `headerShown` ou `headerTransparent`.

### 0.3 Pattern hystérésis existant (A2)

A2 (`docs/chat-ux-refonte/specs/A2.md` §1.4 R23-R26) introduit `deriveHeroCollapsed(scrollY, prev)` : seuils 80↓ / 40↑, fonction pure consommée via `useState` + `onScroll` propagé dans `<ChatSessionSurface>` → `<ChatMessageList>` → `<FlashList>`. A4 réutilise **exactement** le même pattern :

- `deriveTopBarCollapsed(scrollY, prev)` pure (mêmes seuils 80/40 pour cohérence inter-features).
- Le même `onListScroll` du screen (`[sessionId].tsx:114-117`) pilote DEUX states : `heroCollapsed` (A2) ET `topBarCollapsed` (A4).
- Aucune nouvelle propagation `onScroll` — c'est déjà fait par A2 (déjà committé `7f6f0aa2`).

### 0.4 Décision animation — Animated RN (pas Reanimated)

Pour la transition expanded↔collapsed (height interpolation), `Animated` (de `react-native`) est suffisant. Reanimated est **réservé** aux gestures worklet-driven (A2 pinch-zoom). Justification :

- Cohérence avec `ImageFullscreenModal.tsx` qui utilise déjà `Animated` RN pour PanResponder.
- Animation = single-property height (`expandedHeight ↔ collapsedHeight`) — pas de gesture, pas de scroll-driven worklet → driver JS suffit (60Hz acceptable pour transition 200ms ponctuelle).
- ZÉRO worklet à debugger.
- KISS — pas de scope creep Reanimated.

Driver = **`useNativeDriver: false`** (height anime un layout property, RN ne supporte pas height/width sur native driver). Acceptable car l'animation est PONCTUELLE (1× par franchissement de seuil), pas continue.

### 0.5 Reduced motion

`useReducedMotion()` (`shared/ui/hooks/useReducedMotion.ts`) est déjà adopté (voir A2/A3/C4). Quand `true` → snap (Animated.timing duration: 0). Pinch-zoom A2 reste actif (gesture functional). Pour A4 : pure cosmetic motion → respect total. WCAG 2.3.3.

### 0.6 A11y

- `<ChatHeader>` rendu en collapsed mode garde son `accessibilityRole` global implicite (header element). Les actions Pressables conservent labels + roles inchangés.
- En collapsed mode, le titre `<Text numberOfLines={1}>` est tronqué visuellement mais reste lu en entier par VoiceOver (`accessibilityLabel` n'est pas borné par `numberOfLines`).
- L'`<ExpertiseBadge>` est rendu en expanded uniquement (caché en collapsed). Pas de perte info : badge re-apparaît quand l'utilisateur scroll up.
- AI-badge EU AI Act Art.50 (`testID="ai-disclosure-badge"`) **DOIT rester visible et tappable même en collapsed mode** — exigence légale (cf. `docs/legal/AI_DISCLOSURE.md`). C'est une contrainte explicite A4 (R10).
- Boutons actions (close / summary / audio-toggle) restent visibles + tappables en collapsed mode (UX cohérence : actions principales toujours accessibles).
- WCAG 2.5.5 hit target : container collapsed = 40dp height MINIMUM. Boutons ronds restent 36×36dp (`space['9']`) — accepté pour toolbar mini-bar (convention iOS HIG 32×32 button minimum).

## 1. Spec — format EARS

### 1.1 Helper `deriveTopBarCollapsed(scrollY, prev)`

- **R1** : When `deriveTopBarCollapsed(scrollY, previousCollapsed=false)` is called with `scrollY < 80`, the function SHALL return `false`.
- **R2** : When `deriveTopBarCollapsed(scrollY, previousCollapsed=false)` is called with `scrollY >= 80`, the function SHALL return `true`.
- **R3** : When `deriveTopBarCollapsed(scrollY, previousCollapsed=true)` is called with `scrollY >= 40`, the function SHALL return `true`.
- **R4** : When `deriveTopBarCollapsed(scrollY, previousCollapsed=true)` is called with `scrollY < 40`, the function SHALL return `false`.
- **R5** : The module SHALL export the constants `TOP_BAR_COLLAPSE_THRESHOLD = 80` and `TOP_BAR_EXPAND_THRESHOLD = 40`.
- **R6** : The function SHALL be pure (no `useState`, no `useEffect`, no side effect).
- **R7** : The function SHALL be stable across re-renders — same input produces same output, no internal stateful memoization.

### 1.2 Composant `<CollapsibleTopBar>`

- **R8** : When `<CollapsibleTopBar collapsed={false}>` is rendered with valid `<ChatHeader>` props, the component SHALL render `<ChatHeader>` with `collapsed={false}` prop forwarded AND wrap it in an `Animated.View` whose `height` matches the expanded height (≥ 72dp).
- **R9** : When `<CollapsibleTopBar collapsed>` is rendered, the component SHALL render `<ChatHeader>` with `collapsed` prop forwarded AND wrap it in an `Animated.View` whose `height` matches the collapsed height (≤ 48dp).
- **R10** : The `<CollapsibleTopBar>` SHALL render the AI disclosure badge in BOTH expanded and collapsed states (the `<ChatHeader>` already enforces this when `onOpenAiDisclosure` is provided — A4 must not strip the prop). EU AI Act Article 50.
- **R11** : The `<CollapsibleTopBar>` SHALL transition the `Animated.Value` via `Animated.timing` over 200ms with `useNativeDriver: false` when the `collapsed` prop changes.
- **R12** : The `<CollapsibleTopBar>` SHALL respect `useReducedMotion()`. When `useReducedMotion()` returns `true`, the transition SHALL be a snap (duration: 0 or `setValue` immediately) — no animation.
- **R13** : The `<CollapsibleTopBar>` SHALL NOT introduce new i18n keys — it only forwards props to `<ChatHeader>`.
- **R14** : The `<CollapsibleTopBar>` SHALL accept the same set of props as `<ChatHeader>` plus `collapsed?: boolean` (default `false`). The component SHALL NOT consume any other extra prop.
- **R15** : The `<CollapsibleTopBar>` SHALL set a stable `testID="collapsible-top-bar"` on the root `Animated.View` for testability.

### 1.3 `<ChatHeader>` accepts new `collapsed` prop

- **R16** : `<ChatHeader>` SHALL accept a new optional prop `collapsed?: boolean` (default `false`). Backward-compatible — existing call sites without the prop continue to render the expanded layout.
- **R17** : When `<ChatHeader collapsed>` is rendered, the title `<Text>` SHALL use `fontSize.base` (16sp) instead of `fontSize['2xl']` (24sp). `numberOfLines={1}` remains.
- **R18** : When `<ChatHeader collapsed>` is rendered, the `<ExpertiseBadge>` SHALL NOT be rendered (hidden).
- **R19** : When `<ChatHeader collapsed>` is rendered, the AI disclosure badge (if `onOpenAiDisclosure` is provided) SHALL remain rendered AND tappable (EU AI Act Art.50 — see R10).
- **R20** : When `<ChatHeader collapsed>` is rendered, the action buttons (close, summary, audio-toggle) SHALL remain rendered AND tappable.
- **R21** : The new prop SHALL be type-checked — `collapsed` typed as `boolean | undefined`. No `as any`. No `eslint-disable`.

### 1.4 Coupling FE / no BE change

- **R22** : The FE SHALL pass `topBarCollapsed` state from screen to `<CollapsibleTopBar>` via prop drilling — NO global store / context introduced for A4.
- **R23** : The screen `[sessionId].tsx` SHALL maintain TWO separate states : `heroCollapsed` (A2, kept) AND `topBarCollapsed` (A4, new). The same `onListScroll` callback updates BOTH via `deriveHeroCollapsed` and `deriveTopBarCollapsed` respectively.
- **R24** : The backend SHALL NOT be modified by A4. Zero changes to BE types, routes, or contracts.

### 1.5 Out-of-scope (explicite)

- Per-feature feature flag (`*_ENABLED`) — doctrine `feedback_no_feature_flags_prelaunch`.
- Hide top bar entirely on deep scroll.
- Animating GlassCard.intensity during collapse.

### 2.2 Helpers — `useCollapsibleTopBar.ts`

```ts
export const TOP_BAR_COLLAPSE_THRESHOLD = 80;
export const TOP_BAR_EXPAND_THRESHOLD = 40;

/**
 * Pure scroll-collapse computation with hysteresis ; state owned by caller.
 *
 * - Once expanded (previousCollapsed=false), collapses only at scrollY >= 80.
 * - Once collapsed (previousCollapsed=true), re-expands only at scrollY < 40.
 *
 * Mirrors deriveHeroCollapsed (A2) for inter-feature consistency.
 */
export function deriveTopBarCollapsed(scrollY: number, previousCollapsed: boolean): boolean {
  if (previousCollapsed) return scrollY >= TOP_BAR_EXPAND_THRESHOLD;
  return scrollY >= TOP_BAR_COLLAPSE_THRESHOLD;
}
```

### 2.3 Composant — `<CollapsibleTopBar>`

```tsx
import React, { useEffect, useRef } from 'react';
import { Animated } from 'react-native';

import { ChatHeader, type ChatHeaderProps } from '@/features/chat/ui/ChatHeader';
import { useReducedMotion } from '@/shared/ui/hooks/useReducedMotion';

export const COLLAPSIBLE_TOP_BAR_EXPANDED_HEIGHT = 80;
export const COLLAPSIBLE_TOP_BAR_COLLAPSED_HEIGHT = 44;
export const COLLAPSIBLE_TOP_BAR_ANIM_DURATION_MS = 200;

interface CollapsibleTopBarProps extends ChatHeaderProps {
  readonly collapsed?: boolean;
}

export const CollapsibleTopBar = React.memo(({ collapsed = false, ...headerProps }: CollapsibleTopBarProps) => {
  const reducedMotion = useReducedMotion();
  const heightValue = useRef(
    new Animated.Value(collapsed ? COLLAPSIBLE_TOP_BAR_COLLAPSED_HEIGHT : COLLAPSIBLE_TOP_BAR_EXPANDED_HEIGHT),
  ).current;

  useEffect(() => {
    const target = collapsed
      ? COLLAPSIBLE_TOP_BAR_COLLAPSED_HEIGHT
      : COLLAPSIBLE_TOP_BAR_EXPANDED_HEIGHT;
    if (reducedMotion) {
      heightValue.setValue(target);
      return;
    }
    Animated.timing(heightValue, {
      toValue: target,
      duration: COLLAPSIBLE_TOP_BAR_ANIM_DURATION_MS,
      useNativeDriver: false,
    }).start();
  }, [collapsed, reducedMotion, heightValue]);

  return (
    <Animated.View testID="collapsible-top-bar" style={{ height: heightValue, overflow: 'hidden' }}>
      <ChatHeader {...headerProps} collapsed={collapsed} />
    </Animated.View>
  );
});
CollapsibleTopBar.displayName = 'CollapsibleTopBar';
```
