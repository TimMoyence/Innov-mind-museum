**Ce qui EXISTE déjà** : un mécanisme de citations inline `[n]` + bottom-sheet preview, ancré sur des sources vérifiées par grounding substring NFKC.

### 0.2 Ce qui MANQUE — gap A6 (findings.md P11)

Findings.md ligne 175 : « P11 — Pas de signal confiance/source sur claims AI — absent — UFR-013 doctrine pas surfacée UI » et tracking.md ligne 65 : « Catalogue musée vs IA — UFR-013 surfacée UI ».

Le marker `[n]` existant N'EXPOSE PAS :

1. **La provenance lisible humainement** (catalog musée vs Wikidata vs web vs IA générale) — l'utilisateur voit `[1]` `[2]` `[3]` sans distinction visuelle entre une citation d'un catalogue B2B Musaium et un résultat web Brave.
2. **Le niveau de confiance** — `CitationSource.confidence` (0..1) existe en données mais n'est jamais affiché. Aucun badge `High / Medium / Low`.
3. **Le cas "AI knowledge only"** — quand le LLM répond sans aucune source citée (sources absente OU vide), il n'y a AUCUN signal visuel pour avertir l'utilisateur que la réponse est issue de la connaissance générale du modèle (cas le plus risqué pour UFR-013). Le bubble apparaît identique à un bubble bien sourcé.

A6 vient combler ce trio : surfacer **chips provenance + confidence + cas "AI-only"** au bas de chaque bubble assistant.

### 0.3 Sources métier disponibles (mapping confirmé)

Vérifié par grep du BE — `CitationSource.type` peut prendre **exactement 4 valeurs** côté union TS + Zod. Les chips A6 doivent donc agréger ces 4 types vers 3 familles UI lisibles (decision §2.4) :

| `CitationSource.type` (BE) | Famille UI A6 | Tooltip / a11y label en | Trust hint |
|---|---|---|---|
| `museum-catalog` | **Museum catalogue** | "From the museum catalogue" | Highest (B2B curated) |
| `wikidata` | **Reference database** | "From Wikidata reference database" | High (linked-data) |
| `commons` | **Reference database** | "From Wikimedia Commons" | High (curated media) |
| `web` | **Web result** | "From a web search result" | Medium (open web) |
| *(aucune source — `sources` vide ou absent)* | **AI knowledge** | "Based on AI general knowledge — verify before trusting" | Low (model-only) |

Note : `commons` et `wikidata` partagent la même famille UI ("Reference database") car les deux sont issus de l'écosystème Wikimedia / linked-data ; la distinction granulaire reste accessible via la bottom-sheet preview existante (qui montre le `title` exact). Évite la surcharge de 5 chips si une réponse cite 4 sources de types différents.

### 0.4 Confidence — données disponibles + heuristique proposée

- **CitationSource.confidence** : optionnel `number ∈ [0..1]` — populé seulement quand le BE judge l'a dérivé (cf. JSDoc `chat.types.ts:228` « may be absent for KB-direct sources »). Aucun calcul agrégé bubble-level n'existe.
- **detectedArtwork.confidence** : champ orthogonal — confiance d'identification de l'œuvre, pas confiance de la réponse globale.

**Heuristique A6 retenue (FE pure, dérivée de `metadata`)** — `computeConfidenceLevel(metadata): 'high' | 'medium' | 'low'` :

```
if (metadata.sources?.some(s => s.type === 'museum-catalog')) → 'high'
else if (metadata.sources?.some(s => (s.confidence ?? 0) >= 0.8)) → 'high'
else if (metadata.sources && metadata.sources.length > 0) → 'medium'
else → 'low'   // pas de sources : AI knowledge only
```

Trade-off accepté : c'est une heuristique FE-only — le BE pourrait l'exposer (`metadata.confidenceLevel?: 'high'|'medium'|'low'`) plus tard si on veut un audit cross-channel. Pour A6 V1, on garde FE-only pour ne pas réintroduire un round-trip BE→FE-types. Cf. Open Q2.

### 0.5 UI — chip cluster bottom-of-bubble (pas inline numbering)

Décidé d'après doctrine UFR-013 ("surfacée UI ≠ intrusive numbering") et pattern Smashing 2026 cité findings.md §3.1 ligne 118 ("Visual confidence/provenance indicators") :

- **Placement** : juste sous le texte de la bubble, AU-DESSUS de `<TtsSection>`, AU-MEME niveau visuel que `sourcesRow` actuel (`ChatMessageBubble.tsx:96`). Concrètement, `<CitationChips>` remplace ou COMPLÈTE `<View style={styles.sourcesRow}>` (cf. design §2.6 pour cohabitation).
- **Cardinalité** : 1 chip "confidence" obligatoire (couleur DS) + 0..3 chips "provenance" (déduplication par famille UI). Total ≤ 4 chips.
- **Tap** : sur un chip provenance → ouvre la bottom-sheet preview existante (`SourceCitation`'s Modal) en filtrant la liste sur les sources de cette famille. Sur le chip confidence → opens i18n disclosure popover (cf. §2.7).
- **Pas d'inline numbering** : on PEUT garder `[n]` inline pour l'instant (cohabitation), ou les supprimer en T5 (cf. Open Q3 — décision deferred).

### 0.6 i18n existant + clés A6 à introduire

- Clés existantes `chat.sources.*` : déjà 5 strings 8 locales (`viewSource, openLink, quoteFromSource, closeSheet, noQuote`).
- Aucune clé `chat.citation*` ou `chat.confidence*` au moment de l'audit (grep `chat.confidence\|chat.citation` sur 8 locales → 0 résultat).
- Clés A6 à ajouter (sous `chat.citation.*` — nouveau namespace pour éviter collision avec `chat.sources.*` existant) :

## 1. Spec — format EARS

### 1.1 Frontend — composant `<CitationChips>` (cluster)

- **R1** : When an assistant message is rendered AND it is NOT in `isStreaming` state, the system SHALL render a `<CitationChips>` row at the bottom of the assistant bubble (below `<StreamingBody>`, above `<TtsSection>`).
- **R2** : When `<CitationChips>` is rendered, the system SHALL display EXACTLY ONE confidence chip whose label, color, and accessibility text are derived from `computeConfidenceLevel(metadata)` per §0.4.
- **R3** : When `<CitationChips>` is rendered AND `metadata.sources` is non-empty, the system SHALL aggregate `sources[].type` into the 3 UI families (`museum-catalog → museum-catalog`, `wikidata|commons → reference-db`, `web → web`) and render at most 3 provenance chips (deduplicated by family).
- **R4** : When `<CitationChips>` is rendered AND `metadata.sources` is undefined OR empty (length 0), the system SHALL render EXACTLY ONE provenance chip with family `ai-knowledge` (the "AI knowledge only" doctrine chip, cf. UFR-013 surface).
- **R5** : When `<CitationChips>` is rendered for a user message (`role === 'user'`) OR for a streaming/sending placeholder, the system SHALL NOT render any chip (no-op return `null`).
- **R6** : The `<CitationChips>` order SHALL be deterministic : confidence chip FIRST (leftmost in LTR locales) followed by provenance chips ordered by family enum index `['museum-catalog', 'reference-db', 'web', 'ai-knowledge']`.
- **R7** : The `<CitationChips>` SHALL wrap onto multiple lines if needed (no horizontal scroll) — `flexWrap: 'wrap'`, `gap: space['1']`.

### 1.2 Frontend — composant `<CitationChip>` (atomique)

- **R8** : The `<CitationChip>` SHALL be a `<Pressable>` with rounded shape (`borderRadius: semantic.input.radius`), border 1px, and padding `paddingHorizontal: space['1.5']` / `paddingVertical: space['0.5']`.
- **R9** : The `<CitationChip>` for kind `'confidence'` SHALL receive its background and label color from the design tokens map :
  - high → border + text `theme.success`, background `theme.successBackground`
  - medium → border + text `theme.warningText`, background `theme.warningBackground`
  - low → border + text `theme.danger`, background `theme.errorBackground`
- **R10** : The `<CitationChip>` for kind `'provenance'` SHALL render an Ionicons icon next to the label :
  - `museum-catalog` → `business-outline`
  - `reference-db` → `library-outline`
  - `web` → `globe-outline`
  - `ai-knowledge` → `sparkles-outline`
- **R11** : The `<CitationChip>` SHALL NEVER use Unicode emoji (doctrine `feedback_no_unicode_emoji`). Ionicons only.
- **R12** : When `<CitationChip kind="provenance">` is tapped AND `onPress` is provided, the system SHALL invoke `onPress(family)` so the parent can open the source preview filtered on that family.
- **R13** : When `<CitationChip kind="confidence">` is tapped AND `onPress` is provided, the system SHALL invoke `onPress(level)` so the parent can open a disclosure popover explaining the level.
- **R14** : The `<CitationChip>` SHALL set `accessibilityRole="button"` and `accessibilityLabel` to the i18n label (e.g. `chat.citation.family.museum-catalog` resolves to "Museum catalogue") regardless of whether `onPress` is provided.
- **R15** : The `<CitationChip>` SHALL set `accessibilityHint` to `chat.citation.chip.a11y_hint` ("Tap to see the source details") when `onPress` is provided ; SHALL omit hint when not pressable.

### 1.3 Backend — exposer le typage déjà présent (validation contract)

- **R16** : The type alias `CitationSourceType` in `museum-backend/src/modules/chat/domain/chat.types.ts` SHALL remain the exact union `'wikidata' | 'web' | 'museum-catalog' | 'commons'`. Any A6 change to this enum (e.g. adding `'ai-knowledge'`) is OUT-OF-SCOPE — `ai-knowledge` is a FE-only synthetic family, NEVER persisted nor returned by the BE.
- **R17** : `ChatAssistantMetadata` SHALL continue to expose `sources?: CitationSource[]` as today. A6 introduces NO new BE field (no `confidenceLevel`, no `provenanceFamily`). All derived UI semantics live in `museum-frontend/features/chat/application/citations.ts` (new pure module).
- **R18** : When `metadata.sources` is set with at least one entry, the test contract SHALL assert that the FE selector `selectChipModelsForMessage(metadata)` returns a `confidence` chip whose level matches the §0.4 heuristic + a deduplicated provenance chip list mapped via the §0.3 table.

### 1.4 Coupling FE↔BE + telemetry

- **R19** : The FE SHALL NOT throw when `metadata.sources` is missing (legacy persisted messages OR refusal paths — backward-compat as NFR8) — it renders the `ai-knowledge` chip per R4 instead.
- **R20** : The FE SHALL log (`console.debug` in dev, Sentry breadcrumb in prod) one `citation_chips_rendered` event per assistant message rendered with non-empty chips, payload `{ family_count: number; confidence_level: 'high'|'medium'|'low'; has_sources: boolean }`. NO source URLs / titles / quotes emitted (NFR5 PII safety).
- **R21** : The FE SHALL fall-open : if `computeConfidenceLevel` throws unexpectedly (shouldn't happen — pure function), the component SHALL return `null` (no chips) rather than crash the bubble. Defensive guard wraps the helper in `try/catch` returning `null` on error.

### 1.5 i18n

- **R22** : The 8 shipped locales (ar, de, en, es, fr, it, ja, zh) SHALL contain the 10 new keys listed in §0.6 under `chat.citation.*`.
- **R23** : Every `chat.citation.family.*` and `chat.citation.confidence.*` value SHALL be ≤ 28 characters per locale (chip width budget — Mona Lisa stress test in FR/DE).
- **R24** : Every new `chat.citation.*` string SHALL NOT contain any Unicode emoji (regex `/[\u{1F000}-\u{1FFFF}\u{2600}-\u{27BF}]/u` returns false). Doctrine `feedback_no_unicode_emoji`.

### 2.2 Pure helpers — `citations.ts`

```ts
import type {
  CitationSource,
  CitationSourceType,
  ChatUiMessageMetadata,
} from '@/features/chat/application/chatSessionLogic.pure';

export type CitationFamily = 'museum-catalog' | 'reference-db' | 'web' | 'ai-knowledge';
export type ConfidenceLevel = 'high' | 'medium' | 'low';

export const FAMILY_FOR_SOURCE_TYPE: Record<
  CitationSourceType,
  Exclude<CitationFamily, 'ai-knowledge'>
> = {
  'museum-catalog': 'museum-catalog',
  wikidata: 'reference-db',
  commons: 'reference-db',
  web: 'web',
};

export const CITATION_FAMILY_ORDER: readonly CitationFamily[] = [
  'museum-catalog',
  'reference-db',
  'web',
  'ai-knowledge',
] as const;

export type CitationChipModel =
  | { readonly kind: 'confidence'; readonly level: ConfidenceLevel }
  | { readonly kind: 'provenance'; readonly family: CitationFamily; readonly count: number };

export function computeConfidenceLevel(
  metadata: ChatUiMessageMetadata | null | undefined,
): ConfidenceLevel {
  const sources = metadata?.sources;
  if (!sources || sources.length === 0) return 'low';
  if (sources.some((s) => s.type === 'museum-catalog')) return 'high';
  if (sources.some((s) => (s.confidence ?? 0) >= 0.8)) return 'high';
  return 'medium';
}

export function selectChipModelsForMessage(
  metadata: ChatUiMessageMetadata | null | undefined,
): CitationChipModel[] {
  const level = computeConfidenceLevel(metadata);
  const result: CitationChipModel[] = [{ kind: 'confidence', level }];

  const sources = metadata?.sources ?? [];
  if (sources.length === 0) {
    result.push({ kind: 'provenance', family: 'ai-knowledge', count: 0 });
    return result;
  }

  const counts = new Map<CitationFamily, number>();
  for (const s of sources) {
    const family = FAMILY_FOR_SOURCE_TYPE[s.type];
    counts.set(family, (counts.get(family) ?? 0) + 1);
  }

  for (const family of CITATION_FAMILY_ORDER) {
    if (family === 'ai-knowledge') continue;
    const count = counts.get(family);
    if (count && count > 0) {
      result.push({ kind: 'provenance', family, count });
    }
  }

  return result;
}
```

### 2.4 Composant cluster — `<CitationChips>`

```ts
interface CitationChipsProps {
  readonly metadata: ChatUiMessageMetadata | null | undefined;
  readonly onProvenancePress?: (family: CitationFamily) => void;
  readonly onConfidencePress?: (level: ConfidenceLevel) => void;
}

export const CitationChips = React.memo(({ metadata, onProvenancePress, onConfidencePress }: CitationChipsProps) => {
  const models = useMemo(() => {
    try {
      return selectChipModelsForMessage(metadata);
    } catch {
      return [];
    }
  }, [metadata]);

  useEffect(() => {
    if (models.length === 0) return;
    const family_count = models.filter((m) => m.kind === 'provenance').length;
    const confidenceModel = models.find((m) => m.kind === 'confidence');
    const confidence_level = confidenceModel?.level ?? 'low';
    const has_sources = (metadata?.sources?.length ?? 0) > 0;
    console.debug('[A6] citation_chips_rendered', { family_count, confidence_level, has_sources });
  }, [models, metadata]);

  if (models.length === 0) return null;

  return (
    <View style={styles.row}>
      {models.map((m) => (
        <CitationChip
          key={m.kind === 'confidence' ? `c-${m.level}` : `p-${m.family}`}
          model={m}
          onPress={m.kind === 'confidence' ? (() => onConfidencePress?.(m.level)) as never : (() => onProvenancePress?.(m.family)) as never}
        />
      ))}
    </View>
  );
});
```
