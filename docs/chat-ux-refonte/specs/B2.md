## 0. Contexte vérifié

### 0.1 Pain point — où vit la donnée "session précédente"

`findings.md` ligne 177 : « P13 — Pas de conversation resumption — session frag absent — Pattern Pi.ai manqué — **B2** ».

L'app reconnaît déjà les sessions passées (écran `(tabs)/conversations` paginé via `GET /api/chat/sessions`) mais NE pousse JAMAIS le retour-en-arrière vers le home. Quand l'utilisateur revient quelques jours plus tard, le home se contente d'un CTA « Start conversation ». Toute reprise demande à l'utilisateur d'aller dans l'onglet Conversations, scroll, identifier la bonne session — friction inutile sur le scénario museum-mobile principal (« je reviens lundi finir ma balade de samedi »).

Pi.ai « walking back into a conversation » (findings.md §3.1 pattern #5) = bannière proactive au top du home, contextualisée par la dernière œuvre discutée (« Reprendre devant *La Liseuse* de Fragonard ? »).

### 0.2 Données disponibles côté BE — vérification systématique

**Entité `ChatSession`** (`museum-backend/src/modules/chat/domain/session/chatSession.entity.ts:21-75`) — déjà persistée :
- `id: string` UUID
- `museumId?: number | null` (col `museum_id`) — l'identifiant canonique, OK pour matcher « même musée »
- `museumName?: string | null` — libellé affichable
- `coordinates?: { lat: number; lng: number } | null`
- `visitContext?: VisitContext | null` (jsonb) — contient `artworksDiscussed: VisitedArtwork[]` (chat.types.ts:79), chaque entrée portant `title?`/`artist?`/`messageId`/`discussedAt`
- `updatedAt: Date` — utilisé déjà comme tri par `listSessions` repository (sql `ORDER BY updatedAt DESC`)
- `intent: ChatSessionIntent` — `'default' | 'walk'` (filtrer les walks ? cf. §0.7 décision)
- `purgedAt?: Date | null` — sessions purgées GDPR à exclure

**Endpoint listSessions** (`GET /api/chat/sessions`) — pagination cursor existante (`chat-session.route.ts:42-52`). Limite max 50. Réponse `ListSessionsResult` (`chat.service.types.ts:80-103`) expose actuellement :
```
{ id, locale, museumMode, title, museumName, createdAt, updatedAt, intent, preview?, messageCount }
```
**`museumId` n'est PAS exposé** dans la réponse, malgré sa présence sur l'entité. **`visitContext` non plus.** C'est le gap principal pour B2 — il faut savoir, pour chaque session listée, à quel `museumId` numérique elle appartient (matcher) ET quelle est la dernière œuvre discutée (titre du banner).

Mapping `mapSessionToDashboardCard` (`features/chat/domain/dashboard-session.ts:38-61`) — mais ce mapping est destiné à l'écran `(tabs)/conversations`, pas réutilisable tel quel pour la bannière.

Le seul endroit où un `museumId` est connu côté FE est : (a) `useStartConversation()` qui peut le recevoir en param (`StartConversationOptions.museumId`), (b) `useGeofencePreCache({ latitude, longitude })` qui pré-cache via lat/lng. (c) Sur l'écran `(tabs)/museums` map. **Aucun store global « currentMuseumId »** vérifié par grep (`grep -rn "currentMuseumId\|activeMuseum" features/` → 0 résultats applicatifs).

### 0.4 Pattern de placement — où vit la bannière

Findings.md §2 + tracking.md ligne 67 : « banner sur l'écran d'accueil ou screen chat list propose ... ». Décision dispatcher (input utilisateur, point §3 du prompt) :

**Home screen UNIQUEMENT** (`museum-frontend/app/(tabs)/home.tsx`), placé entre `<GlassCard heroCard>` (BrandMark + intro) et `<DailyArtCard>`, c.-à-d. **au-dessus du primary CTA et au-dessus du daily art** — c'est le scroll-position qui maximise la visibilité (Pi.ai pattern : la bannière de reprise est la première chose vue post-app-launch).

Justification non-duplication onglet Conversations :
1. L'onglet `(tabs)/conversations` montre déjà TOUTES les sessions, triées par updatedAt → la « reprise » y est implicite par la liste.
2. Doubler la bannière dans `conversations` dépriserait visuellement la liste elle-même.
3. KISS : un seul point d'entrée, pas deux à maintenir.

### 0.5 Critères d'affichage — décisions

| Critère | Verdict | Justification |
|---|---|---|
| Fenêtre temporelle | **< 7 jours** (168h) calculé en différence avec `now()` sur `updatedAt` ISO string | Pattern Pi.ai documenté findings.md §3.1#5. < 7j = fraîcheur perceptive « ma visite récente », > 7j = visite passée, bannière intrusive. |
| Sélection de la session | **La PLUS RÉCENTE par `updatedAt` DESC**, première page (limit=10) suffit | Match utilisateur attendu — « ma dernière conversation ». Multi-session reprise = hors scope V1 (Open Q4). |
| Match musée | **Match par `museumId === currentMuseumId` si BOTH defined, sinon par `museumName === currentMuseumName` (case-insensitive trim), sinon fallback : afficher quand même** | Cf. décision Open Q1 — V1 affiche la bannière même hors musée (« reprendre votre dernière visite ? ») ; le matching exact strict gating museum est V1.1. Doctrine `feedback_hybrid_product_philosophy` : pas de gating dur sur la géoloc, sinon UX décevante hors-musée. |
| Sessions vides | **Exclues** — `messageCount === 0` filtre out | Pas de reprise d'une session jamais utilisée. |
| Sessions purgées | **Exclues côté BE** — `WHERE purgedAt IS NULL` déjà appliqué (vérifier dans chat-purge cron) | GDPR : pas de reprise d'une session legalement effacée. |
| Sessions walk | **Incluses V1** | Mode balade = reprise pertinente (« continuer votre balade ? »). Toggle V1.1 si signal négatif (Open Q5). |
| Dismiss user | **24 h via AsyncStorage** | Pattern §3.6 prompt. Clé `settings.resumption_banner_dismissed_until` = ISO timestamp jusqu'à laquelle on hide. Au-delà, refichier. |
| Bannière unique par session | **Non-stickering** — si user dismiss, le dismiss porte sur le user, pas sur la session. La même bannière peut réapparaître pour la même session après expiration du dismiss + dans la fenêtre 7j. | KISS V1. Per-session dismiss = V1.1 (Open Q3). |

### 0.6 Format de la bannière

Card horizontale, structure :
```
┌────────────────────────────────────────────────────┐
│  [thumb 48dp]   Continue your conversation         │
│                 about La Liseuse                   │  [X]
│                 Louvre — 2 days ago                │
└────────────────────────────────────────────────────┘
       (whole card pressable = navigate to chat session)
```

- **Thumb** : si la dernière `VisitedArtwork.title` correspond à un message avec `image?.url` dans la session, on n'a PAS de URL d'image dans le BE list response (R30 §1.4 décision V1). **Fallback iconique** : Ionicons `images-outline` 32dp dans un cercle `theme.surfaceSecondary`. (Image réelle de la dernière œuvre = V1.1 Open Q6.)
- **Title** : i18n `chat.resumption.title_with_artwork` interpolé avec `{{title}}` quand last artwork disponible, sinon `chat.resumption.title_fallback` (« Continue your last conversation »).
- **Subtitle** : `{{museumName}} — {{timeAgo}}` (e.g. « Louvre — 2 days ago »), `timeAgo` via i18n keys `chat.resumption.time_ago.{just_now,minutes,hours,days}` interpolés (no third-party date-fns introduction, NFR7).
- **Dismiss X** : Ionicons `close` 20dp, hit target ≥ 44dp via `hitSlop`. **`accessibilityRole='button'` séparé** du card root, label `chat.resumption.a11y_dismiss`. Tap = écrit `settings.resumption_banner_dismissed_until` + state local cache → bannière disparaît immédiatement.
- **Card press** : navigate `router.push(`/(stack)/chat/${sessionId}`)` — réutilise le router déjà importé par `useStartConversation`. **AUCUN `createSession` appelé** (la session existe déjà).

## 1. Spec — format EARS

### 1.1 Hook `useResumableSession()` — sélection + visibility logic

- **R1** : `useResumableSession()` SHALL return an object `{ session: ResumableSession | null; isLoading: boolean; dismiss: () => Promise<void> }`.
- **R2** : On mount, the hook SHALL call `chatApi.listSessions({ limit: 10 })` exactly once. It SHALL NOT re-fetch on every render.
- **R3** : The hook SHALL filter the returned `sessions` array keeping ONLY items satisfying ALL of :
  - `messageCount > 0`
  - `Date.now() - new Date(updatedAt).getTime() < SEVEN_DAYS_MS` where `SEVEN_DAYS_MS = 7 * 24 * 60 * 60 * 1000`.
- **R4** : Among filtered items, the hook SHALL pick the one with the most recent `updatedAt` (the BE already returns DESC ordering by `updatedAt`, but the hook SHALL NOT assume ordering — it derives the max explicitly).
- **R5** : When NO item satisfies R3, `session` SHALL be `null`.
- **R6** : Before resolving `session`, the hook SHALL read `storage.getItem('settings.resumption_banner_dismissed_until')`. If the value is a parseable ISO string `T` AND `Date.now() < T.getTime()`, the hook SHALL return `session: null` (dismissed-until window still active).
- **R7** : When the storage read fails or the value is unparseable, the hook SHALL treat it as "not dismissed" and proceed with normal selection (no throw).
- **R8** : The hook SHALL expose `dismiss()` which (a) writes `settings.resumption_banner_dismissed_until` with `(Date.now() + 24*60*60*1000).toISOString()`, (b) sets the local `session` state to `null` synchronously (optimistic UI), (c) does NOT re-fetch.
- **R9** : `ResumableSession` SHALL have shape `{ id: string; museumId: number | null; museumName: string | null; lastArtworkTitle: string | null; updatedAt: string }`. No other field is required for V1.
- **R10** : The hook SHALL log `console.debug('[B2] resumable_session_shown', { has_artwork_title: boolean; has_museum: boolean; age_hours: number })` when `session !== null` on first resolution. No PII (no title text, no museumName, no sessionId) is emitted.
- **R11** : The hook SHALL NOT throw on any API error — `listSessions` failure resolves `session: null` and a `console.debug('[B2] resumable_session_fetch_failed', { reason })` (no error stack to avoid leaking).
- **R12** : The hook SHALL constant-export `RESUMPTION_BANNER_DISMISS_STORAGE_KEY = 'settings.resumption_banner_dismissed_until'` and `RESUMPTION_BANNER_DISMISS_DURATION_MS = 24*60*60*1000` and `RESUMPTION_BANNER_WINDOW_MS = 7*24*60*60*1000` for testability.

### 1.2 Composant `<ConversationResumptionBanner>` — rendu

- **R13** : When `<ConversationResumptionBanner session={null}>` is rendered, the component SHALL return `null` (no DOM, no layout).
- **R14** : When `<ConversationResumptionBanner session={session}>` with non-null session is rendered, the component SHALL render a `<Pressable testID="conversation-resumption-banner">` at the root with `accessibilityRole="button"`.
- **R15** : The root `accessibilityLabel` SHALL be interpolated from `chat.resumption.a11y_label_with_artwork` when `session.lastArtworkTitle` is non-empty AND `session.museumName` is non-empty, else from `chat.resumption.a11y_label_fallback`.
- **R16** : The root `accessibilityHint` SHALL be set to `chat.resumption.a11y_hint`.
- **R17** : The component SHALL render a title `<Text testID="conversation-resumption-title">` whose content is `t('chat.resumption.title_with_artwork', { title: session.lastArtworkTitle })` when `lastArtworkTitle` is non-empty, else `t('chat.resumption.title_fallback')`.
- **R18** : The component SHALL render a subtitle `<Text testID="conversation-resumption-subtitle">` whose content is `t('chat.resumption.subtitle_with_museum', { museumName, timeAgo })` when `museumName` is non-empty, else `t('chat.resumption.subtitle_no_museum', { timeAgo })`. `timeAgo` is computed by a pure helper `formatResumptionTimeAgo(updatedAtIso, now, t)` exported from the same module.
- **R19** : The dismiss button SHALL be a separate `<Pressable testID="conversation-resumption-dismiss">` with its OWN `accessibilityRole="button"` and its own `accessibilityLabel` = `chat.resumption.a11y_dismiss`. It MUST NOT be a child of the card root Pressable in a way that conflates the two tap zones — RN's nested Pressable tap handling MUST register the inner press without triggering the outer press (use `event.stopPropagation()` via inner `onPress`).
- **R20** : Card press SHALL invoke the `onResume(sessionId)` callback with the session id verbatim.
- **R21** : Dismiss press SHALL invoke the `onDismiss()` callback with no argument.
- **R22** : The component SHALL render an Ionicons `images-outline` thumbnail placeholder inside a circular container of background `theme.surface` (V1 placeholder ; real image deferred V1.1 per Open Q6).
- **R23** : The component SHALL NEVER use Unicode emoji (doctrine `feedback_no_unicode_emoji`). Ionicons only.
- **R24** : The component SHALL be `React.memo`'d with default ref equality.

### 1.3 Helper `formatResumptionTimeAgo(updatedAtIso, now, t)` — pure

- **R25** : When `now - updatedAt < 60_000` ms, the helper SHALL return `t('chat.resumption.time_ago.just_now')`.
- **R26** : When `60_000 ms ≤ now - updatedAt < 3_600_000` ms, the helper SHALL return `t('chat.resumption.time_ago.minutes', { count })` where `count = Math.floor(delta / 60_000)`.
- **R27** : When `3_600_000 ms ≤ now - updatedAt < 86_400_000` ms, the helper SHALL return `t('chat.resumption.time_ago.hours', { count })` where `count = Math.floor(delta / 3_600_000)`.
- **R28** : When `now - updatedAt ≥ 86_400_000` ms, the helper SHALL return `t('chat.resumption.time_ago.days', { count })` where `count = Math.floor(delta / 86_400_000)`.
- **R29** : When `updatedAtIso` is invalid (`Number.isNaN(parsed.getTime()) === true`), the helper SHALL return `t('chat.resumption.time_ago.just_now')` as a safe fallback (no throw, no empty string).

### 1.4 BE contract — extension MINIMALE de `listSessions`

- **R30** : `ListSessionsResult.sessions[number]` (defined in `museum-backend/src/modules/chat/useCase/orchestration/chat.service.types.ts`) SHALL gain TWO optional fields :
  - `museumId?: number | null` — mirrors `ChatSession.museumId` directly.
  - `lastArtworkTitle?: string | null` — derived from `session.visitContext?.artworksDiscussed?.[length-1]?.title ?? null`. When the array is empty or absent, the field is `null`.
- **R31** : The mapping `chat-session.service.ts:246-264` SHALL populate the two new fields with NO extra database query (the entity row already loads `museumId` and `visitContext` jsonb). No SQL change, no schema migration.
- **R32** : The OpenAPI spec (`museum-backend/openapi/openapi.yml`) SHALL be updated to add the two fields as optional/nullable in `SessionListItem`. The FE-side generated `openapi.ts` is regenerated via `npm run generate:openapi-types` (T1.5).
- **R33** : The runtime guard `isListSessionsResponseDTO` (`museum-frontend/features/chat/domain/contracts.ts:203-247`) SHALL accept items WITHOUT the new fields (legacy clients/server) — the fields are strictly optional. A field present must be `number | null` for `museumId` and `string | null` for `lastArtworkTitle`. Any other type SHALL fail the guard.

### 1.5 Coupling FE / contract

- **R34** : The FE SHALL NOT crash when the API returns sessions without the new fields (R33 — defensive optional chaining + fallback to `null`).
- **R35** : The hook SHALL NOT call `useStartConversation` — banner navigates to existing sessions via `router.push`, NEVER creates a new session.
- **R36** : The screen `(tabs)/home.tsx` SHALL pass `onResume={(id) => router.push(\`/(stack)/chat/\${id}\`)}` and `onDismiss={dismiss}` from the hook to the component. No global store introduced.

### 1.6 i18n

- **R37** : The 8 shipped locales (ar, de, en, es, fr, it, ja, zh) SHALL contain the 12 new keys listed in §0.8 under `chat.resumption.*`.
- **R38** : Every `chat.resumption.*` string SHALL NOT contain any Unicode emoji (regex `/[\u{1F000}-\u{1FFFF}\u{2600}-\u{27BF}]/u` returns false).
- **R39** : `chat.resumption.title_with_artwork`, `chat.resumption.title_fallback`, `chat.resumption.subtitle_no_museum` per locale SHALL be ≤ 60 characters (visual budget on iPhone SE width minus thumb + dismiss).

### 1.7 Out-of-scope (explicite)

- New endpoint `/api/chat/sessions/last?museumId=X` (deferred — V1 reuses list).
- Server-side filtering by museum context (deferred — FE filters client-side V1).
- Real thumbnail of the last user image / detected artwork (V1.1, Open Q6).
- Per-session dismiss tracking (V1 = 24h global dismiss).
- Multi-session resumption (display up to N banners) — V1.1 if signal supports it.
- Push notification on the day-after (out — distinct feature, V2).
- Auto-detection of "user is back at the same museum" via GPS — deferred until B6's location consent UX lands.
- Walk-mode-only banner (V1 includes both `default` and `walk` intents).
- Animation/transition (V1 = mount/unmount snap — `useReducedMotion()` N/A).

### 2.2 Helpers — `useResumableSession.ts`

```ts
import { useCallback, useEffect, useState } from 'react';

import { chatApi } from '@/features/chat/infrastructure/chatApi';
import { storage } from '@/shared/infrastructure/storage';

export const RESUMPTION_BANNER_DISMISS_STORAGE_KEY = 'settings.resumption_banner_dismissed_until';
export const RESUMPTION_BANNER_DISMISS_DURATION_MS = 24 * 60 * 60 * 1000;
export const RESUMPTION_BANNER_WINDOW_MS = 7 * 24 * 60 * 60 * 1000;

export interface ResumableSession {
  readonly id: string;
  readonly museumId: number | null;
  readonly museumName: string | null;
  readonly lastArtworkTitle: string | null;
  readonly updatedAt: string;
}

export function useResumableSession() {
  const [session, setSession] = useState<ResumableSession | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    let cancelled = false;
    void (async () => {
      try {
        // 1. Dismiss-until gate.
        const dismissedUntilRaw = await storage.getItem(RESUMPTION_BANNER_DISMISS_STORAGE_KEY).catch(() => null);
        if (dismissedUntilRaw) {
          const dismissedUntil = new Date(dismissedUntilRaw).getTime();
          if (!Number.isNaN(dismissedUntil) && Date.now() < dismissedUntil) {
            if (!cancelled) setIsLoading(false);
            return;
          }
        }

        // 2. Fetch.
        const response = await chatApi.listSessions({ limit: 10 });
        if (cancelled) return;

        // 3. Filter + pick max.
        const now = Date.now();
        const eligible = response.sessions.filter(
          (s) => s.messageCount > 0 && now - new Date(s.updatedAt).getTime() < RESUMPTION_BANNER_WINDOW_MS,
        );
        if (!eligible.length) {
          setIsLoading(false);
          return;
        }
        const latest = eligible.reduce((best, current) =>
          new Date(current.updatedAt).getTime() > new Date(best.updatedAt).getTime() ? current : best,
        );

        const next: ResumableSession = {
          id: latest.id,
          museumId: (latest as { museumId?: number | null }).museumId ?? null,
          museumName: latest.museumName ?? null,
          lastArtworkTitle: (latest as { lastArtworkTitle?: string | null }).lastArtworkTitle ?? null,
          updatedAt: latest.updatedAt,
        };
        setSession(next);
        setIsLoading(false);

        console.debug('[B2] resumable_session_shown', {
          has_artwork_title: next.lastArtworkTitle !== null,
          has_museum: next.museumName !== null,
          age_hours: Math.floor((now - new Date(next.updatedAt).getTime()) / 3_600_000),
        });
      } catch (reason) {
        if (!cancelled) {
          setIsLoading(false);
          console.debug('[B2] resumable_session_fetch_failed', { reason: 'fetch-error' });
        }
      }
    })();
    return () => {
      cancelled = true;
    };
  }, []);

  const dismiss = useCallback(async () => {
    setSession(null);
    const until = new Date(Date.now() + RESUMPTION_BANNER_DISMISS_DURATION_MS).toISOString();
    await storage.setItem(RESUMPTION_BANNER_DISMISS_STORAGE_KEY, until);
  }, []);

  return { session, isLoading, dismiss };
}
```

### 2.3 Composant — `<ConversationResumptionBanner>`

```ts
interface ConversationResumptionBannerProps {
  readonly session: ResumableSession | null;
  readonly onResume?: (sessionId: string) => void;
  readonly onDismiss?: () => void;
}

export function formatResumptionTimeAgo(updatedAtIso: string, now: number, t: TFunction): string {
  const updatedAt = new Date(updatedAtIso).getTime();
  if (Number.isNaN(updatedAt)) return t('chat.resumption.time_ago.just_now');
  const delta = now - updatedAt;
  if (delta < 60_000) return t('chat.resumption.time_ago.just_now');
  if (delta < 3_600_000) return t('chat.resumption.time_ago.minutes', { count: Math.floor(delta / 60_000) });
  if (delta < 86_400_000) return t('chat.resumption.time_ago.hours', { count: Math.floor(delta / 3_600_000) });
  return t('chat.resumption.time_ago.days', { count: Math.floor(delta / 86_400_000) });
}

export const ConversationResumptionBanner = React.memo(
  ({ session, onResume, onDismiss }: ConversationResumptionBannerProps) => {
    const { t } = useTranslation();
    const { theme } = useTheme();
    if (!session) return null;

    const hasArtwork = Boolean(session.lastArtworkTitle && session.lastArtworkTitle.length);
    const hasMuseum = Boolean(session.museumName && session.museumName.length);
    const timeAgo = formatResumptionTimeAgo(session.updatedAt, Date.now(), t);

    const title = hasArtwork
      ? t('chat.resumption.title_with_artwork', { title: session.lastArtworkTitle })
      : t('chat.resumption.title_fallback');
    const subtitle = hasMuseum
      ? t('chat.resumption.subtitle_with_museum', { museumName: session.museumName, timeAgo })
      : t('chat.resumption.subtitle_no_museum', { timeAgo });
    const a11yLabel =
      hasArtwork && hasMuseum
        ? t('chat.resumption.a11y_label_with_artwork', {
            title: session.lastArtworkTitle,
            museumName: session.museumName,
          })
        : t('chat.resumption.a11y_label_fallback');

    return (
      <Pressable
        testID="conversation-resumption-banner"
        accessibilityRole="button"
        accessibilityLabel={a11yLabel}
        accessibilityHint={t('chat.resumption.a11y_hint')}
        onPress={() => onResume?.(session.id)}
        style={[styles.root, { backgroundColor: theme.cardBackground }]}
      >
        <View style={[styles.thumb, { backgroundColor: theme.surface }]}>
          <Ionicons name="images-outline" size={20} color={theme.textSecondary} />
        </View>
        <View style={styles.content}>
          <Text testID="conversation-resumption-title" style={[styles.title, { color: theme.textPrimary }]} numberOfLines={1}>
            {title}
          </Text>
          <Text testID="conversation-resumption-subtitle" style={[styles.subtitle, { color: theme.textSecondary }]} numberOfLines={1}>
            {subtitle}
          </Text>
        </View>
        <Pressable
          testID="conversation-resumption-dismiss"
          accessibilityRole="button"
          accessibilityLabel={t('chat.resumption.a11y_dismiss')}
          onPress={(e) => {
            e.stopPropagation();
            onDismiss?.();
          }}
          hitSlop={12}
          style={styles.dismiss}
        >
          <Ionicons name="close" size={20} color={theme.textSecondary} />
        </Pressable>
      </Pressable>
    );
  },
);
ConversationResumptionBanner.displayName = 'ConversationResumptionBanner';
```
