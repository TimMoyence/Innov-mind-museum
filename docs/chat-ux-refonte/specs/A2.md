### 0.3 Pairing image user ↔ detectedArtwork — heuristique

L'œuvre primaire d'une session se résout en deux étapes :

1. Itérer les messages dans l'ordre chronologique (`sortByTime` existe déjà — `chatSessionLogic.pure.ts:109`).
2. Trouver le PREMIER `role === 'user'` avec `image?.url` non-vide. C'est **l'image primaire** affichée dans la hero card.
3. Trouver, parmi les messages assistant venant APRÈS cette user-image (chronologiquement), le PREMIER message dont `metadata.detectedArtwork?.title` est défini. C'est la **métadonnée primaire**.
4. Si étape 3 échoue mais étape 2 a réussi → fallback "anonymous artwork" : hero card rendue avec seulement l'image (titre = i18n `chat.artworkHero.untitled`).
5. Si étape 2 échoue → pas de hero card du tout (composant retourne `null`).

Cette heuristique tient compte de la doctrine `feedback_check_tests_before_bug_classification` : la mémoire `project_c2_ai_side_only` confirme « Multi-image upload explicitly rejected 2026-05-08 », donc l'image primaire == première image uploadée, pas besoin de logique multi-image.

Cas particuliers vérifiés :

- Plusieurs images user dans le même thread : seule la PREMIÈRE devient hero. Les suivantes restent inline dans leur bulle. Cohérence avec UX "balade d'une œuvre".
- Une seule image, deux assistant messages (premier = guardrail refus, second = vraie réponse) : on prend `detectedArtwork` du PREMIER non-vide chronologique (refus n'aura pas `detectedArtwork` car guardrail interrompt avant la pipeline LLM → `detectedArtwork` absent → on saute → second message gagne).
- Image user sans détection (LLM ne reconnaît rien) : fallback step 4.

## 1. Spec — format EARS

### 1.1 Hook `useArtworkHero(messages)` — sélection primaire

- **R1** : When `useArtworkHero(messages)` is called with an empty array `[]`, the hook SHALL return `null`.
- **R2** : When `useArtworkHero(messages)` is called with messages where NO `role === 'user'` message has a non-empty `image?.url`, the hook SHALL return `null`.
- **R3** : When `useArtworkHero(messages)` is called with messages containing at least one `role === 'user'` message with `image?.url` non-empty, the hook SHALL return an `ArtworkHeroModel` whose `imageUrl` equals the `image.url` of the FIRST such message (chronological order via `createdAt`).
- **R4** : When `useArtworkHero(messages)` finds a primary user image at index `iUser`, the hook SHALL look for the FIRST `role === 'assistant'` message with `metadata.detectedArtwork?.title` non-empty AND `createdAt >= messages[iUser].createdAt`. If found, `ArtworkHeroModel.title / artist / museum / room / confidence` SHALL be derived from that assistant's `detectedArtwork`.
- **R5** : When `useArtworkHero(messages)` finds a primary user image but NO matching assistant `detectedArtwork` (R4 negative), the hook SHALL return `ArtworkHeroModel` with `imageUrl` set, `title === null`, `artist === null`, `museum === null`, `room === null`, `confidence === null`. (Fallback "untitled artwork" — §0.3 step 4.)
- **R6** : The hook SHALL be a pure derivation memoized via `useMemo([messages])`. No side effects, no fetches.
- **R7** : The hook SHALL skip messages where `role === 'system'`.

### 1.2 Composant `<ArtworkHeroCard>` — collapsed/expanded modes (in-place)

- **R8** : When `<ArtworkHeroCard model={null}>` is rendered, the component SHALL return `null` (no DOM, no layout).
- **R9** : When `<ArtworkHeroCard model={model} collapsed={false}>` is rendered, the component SHALL render a horizontal row with the image thumb (target 56dp circular), the title, the artist (if not null), and the museum-room line (if museum not null).
- **R10** : When `<ArtworkHeroCard model={model} collapsed>` is rendered, the component SHALL render a compact row with the image thumb (target 32dp circular) and the title truncated to one line (`numberOfLines={1}`). The artist + museum-room are NOT rendered in this mode.
- **R11** : The `<ArtworkHeroCard>` SHALL be a `<Pressable>` at the root with `accessibilityRole="button"` and `accessibilityLabel` interpolated from `chat.artworkHero.a11y_label` (or `chat.artworkHero.a11y_label_untitled` when `model.title === null`).
- **R12** : The `<ArtworkHeroCard>` SHALL invoke its `onExpand()` callback when pressed.
- **R13** : When `model.title === null`, the rendered title SHALL be the i18n string `chat.artworkHero.untitled`.
- **R14** : The `<ArtworkHeroCard>` SHALL set `accessibilityHint` to `chat.artworkHero.a11y_hint` when `onExpand` is provided.
- **R15** : The `<ArtworkHeroCard>` SHALL NEVER use Unicode emoji (doctrine `feedback_no_unicode_emoji`). Image + Ionicons (chevron / image-outline placeholder if needed) only.

### 1.3 Composant `<ArtworkHeroModal>` — expanded fullscreen + pinch-zoom

- **R16** : When `<ArtworkHeroModal visible={false} model={...}>` is rendered, the modal SHALL NOT be mounted to the screen (RN `Modal` is `visible={false}`).
- **R17** : When `<ArtworkHeroModal visible model={model}>` is rendered, the modal SHALL display the image fullsize (`resizeMode="contain"`), the title (i18n fallback if null), the artist (if not null), the museum-room line (if museum not null), and a close button.
- **R18** : The close button SHALL invoke `onClose()` on press AND set `accessibilityLabel` to `chat.artworkHero.modal.close`.
- **R19** : The modal SHALL listen to Android hardware-back via `BackHandler.addEventListener('hardwareBackPress', ...)` and invoke `onClose()` (pattern reused from `MuseumSheet.tsx:42`). The listener SHALL be cleaned up on unmount.
- **R20** : The modal SHALL wrap the image in `<GestureDetector gesture={Gesture.Pinch()}>` (from `react-native-gesture-handler`), binding the pinch's `scale` to a `useSharedValue` from `react-native-reanimated` clamped to `[1, 5]` via `useAnimatedStyle`.
- **R21** : When `useReducedMotion()` returns `true`, the pinch handler SHALL stay active (doctrine — `useReducedMotion` only disables decorative motion ; pinch is functional, cf. `useReducedMotion.ts:11` comment).
- **R22** : The modal SHALL NEVER use Unicode emoji. Ionicons `close-circle` for the close button, RN `Image` for the artwork.

### 1.4 Hook `useArtworkHeroCollapse(scrollY)` — scroll-driven mode

- **R23** : When `useArtworkHeroCollapse` is invoked with `scrollY < 80`, the returned `isCollapsed` SHALL be `false`.
- **R24** : When `useArtworkHeroCollapse` is invoked with `scrollY >= 80`, the returned `isCollapsed` SHALL be `true`.
- **R25** : The hook SHALL apply a hysteresis : once collapsed at `>= 80`, it SHALL remain collapsed until `scrollY < 40` (re-expand). Prevents flicker at the threshold.
- **R26** : The hook SHALL be pure — `useState` + `useEffect`-free derivation via `useMemo` on the scroll value passed in.

### 1.5 Coupling FE / contract

- **R27** : The FE SHALL NOT throw when `messages` contains malformed entries (missing `image`, `metadata`, `createdAt`) — defensive guards via optional chaining. The hero card returns `null` rather than crash the screen.
- **R28** : The `<ArtworkHeroCard>` SHALL log (`console.debug`) one `artwork_hero_rendered` event per session render with payload `{ has_title: boolean; has_artist: boolean; has_museum: boolean; collapsed: boolean }`. NO `imageUrl` / artwork ID emitted (NFR4 PII safety — image URLs are signed S3 ephemeral but still excluded).
- **R29** : The FE SHALL pass `collapsed` state from screen to component via prop drilling — NO global store / context introduced for A2.

### 1.6 Backend — pas de changement

- **R30** : `museum-backend/src/modules/chat/domain/chat.types.ts` SHALL NOT be modified by A2. `ChatAssistantMetadata.detectedArtwork` already exposes the needed fields. Any BE change (e.g. surface `imageUrl` in detectedArtwork to dispense pairing) is OUT-OF-SCOPE (cf. Open Q1).

- **R32** : Every `chat.artworkHero.untitled` value SHALL be ≤ 28 characters per locale (mini-collapsed mode width budget on iPhone SE).

### 2.2 Helpers — `useArtworkHero.ts`

```ts
import { useMemo } from 'react';
import type { ChatUiMessage } from '@/features/chat/application/chatSessionLogic.pure';

export interface ArtworkHeroModel {
  readonly imageUrl: string;
  readonly title: string | null;
  readonly artist: string | null;
  readonly museum: string | null;
  readonly room: string | null;
  readonly confidence: number | null;
}

export const ARTWORK_HERO_COLLAPSE_THRESHOLD = 80;
export const ARTWORK_HERO_EXPAND_THRESHOLD = 40;

export function useArtworkHero(messages: ChatUiMessage[]): ArtworkHeroModel | null {
  return useMemo(() => {
    if (!messages.length) return null;
    const sorted = [...messages].sort(
      (a, b) => new Date(a.createdAt).getTime() - new Date(b.createdAt).getTime(),
    );

    const userImageIdx = sorted.findIndex(
      (m) => m.role === 'user' && typeof m.image?.url === 'string' && m.image.url.length > 0,
    );
    if (userImageIdx < 0) return null;

    const imageUrl = sorted[userImageIdx]!.image!.url;
    const userCreatedAt = new Date(sorted[userImageIdx]!.createdAt).getTime();

    const match = sorted.find(
      (m) =>
        m.role === 'assistant' &&
        new Date(m.createdAt).getTime() >= userCreatedAt &&
        typeof m.metadata?.detectedArtwork?.title === 'string' &&
        m.metadata.detectedArtwork.title.length > 0,
    );

    if (!match) {
      return { imageUrl, title: null, artist: null, museum: null, room: null, confidence: null };
    }

    const d = match.metadata!.detectedArtwork!;
    return {
      imageUrl,
      title: d.title ?? null,
      artist: d.artist ?? null,
      museum: d.museum ?? null,
      room: d.room ?? null,
      confidence: d.confidence ?? null,
    };
  }, [messages]);
}

// Pure scroll-collapse computation with hysteresis ; state owned by caller.
export function deriveHeroCollapsed(scrollY: number, previousCollapsed: boolean): boolean {
  if (previousCollapsed) return scrollY >= ARTWORK_HERO_EXPAND_THRESHOLD;
  return scrollY >= ARTWORK_HERO_COLLAPSE_THRESHOLD;
}
```

### 2.4 Composant — `<ArtworkHeroModal>` (pinch-zoom)

```ts
interface ArtworkHeroModalProps {
  readonly visible: boolean;
  readonly model: ArtworkHeroModel | null;
  readonly onClose: () => void;
}

export const ArtworkHeroModal = React.memo(({ visible, model, onClose }: ArtworkHeroModalProps) => {
  const { t } = useTranslation();
  const scale = useSharedValue(1);
  const savedScale = useSharedValue(1);

  useEffect(() => {
    if (!visible) return;
    const sub = BackHandler.addEventListener('hardwareBackPress', () => {
      onClose();
      return true;
    });
    return () => sub.remove();
  }, [visible, onClose]);

  const pinch = Gesture.Pinch()
    .onUpdate((e) => {
      scale.value = Math.min(Math.max(savedScale.value * e.scale, 1), 5);
    })
    .onEnd(() => {
      savedScale.value = scale.value;
    });

  const animatedStyle = useAnimatedStyle(() => ({
    transform: [{ scale: scale.value }],
  }));

  if (!model) return null;

  return (
    <Modal visible={visible} transparent animationType="fade" onRequestClose={onClose}>
      <SafeAreaView style={styles.root}>
        <Pressable
          style={styles.closeButton}
          onPress={onClose}
          accessibilityRole="button"
          accessibilityLabel={t('chat.artworkHero.modal.close')}
          hitSlop={12}
        >
          <Ionicons name="close-circle" size={32} color="#fff" />
        </Pressable>

        <GestureDetector gesture={pinch}>
          <Animated.View style={[styles.imageArea, animatedStyle]}>
            <Image source={{ uri: model.imageUrl }} style={styles.fullImage} resizeMode="contain" />
          </Animated.View>
        </GestureDetector>

        <View style={styles.bottomBar}>
          <Text style={styles.title}>{model.title ?? t('chat.artworkHero.modal.title_fallback')}</Text>
          {model.artist ? <Text style={styles.subtitle}>{model.artist}</Text> : null}
          {model.museum ? (
            <Text style={styles.location}>{[model.museum, model.room].filter(Boolean).join(' — ')}</Text>
          ) : null}
        </View>
      </SafeAreaView>
    </Modal>
  );
});
```

## 5. Open questions

Q1-Q5 à arbitrer avant green-code-agent — non bloquantes pour démarrer T1.1 (hook) ou T2.1 (composant card).

- **Q1 (BE expose `detectedArtwork.imageUrl` ?)** : Actuellement le BE attache `image.url` sur le user message et `detectedArtwork` sur l'assistant message ; le pairing est fait côté FE par recherche du PREMIER user-image post-LLM-match. Si le BE remettait l'`imageUrl` de l'image qui a déclenché la détection DIRECTEMENT dans `detectedArtwork`, le pairing FE serait trivial. **Proposition** : deferred V1.1 — V1 garde le pairing FE-only (zéro changement BE, pas de migration de données legacy).
- **Q2 (Cohabitation avec `<ArtworkCard>` inline bubble ?)** : `<ArtworkCard>` rend la metadata dans la bulle assistant ; `<ArtworkHeroCard>` la pin au top. Les deux affichent les mêmes données → double-affichage perçu. **Proposition** : V1 cohabitation (zéro régression UX), évaluation suppression V1.1 post-feedback utilisateur. Si suppression : `<ArtworkCard>` reste pour œuvres SECONDAIRES (mentions hors-image dans le thread, e.g. "voici une œuvre similaire dans la salle d'à côté").
- **Q3 (Traductions 6 locales non-en/fr)** : Le red-test ne fixe que sentinelles en/fr (§2.7). Les 6 autres locales reçoivent des traductions à valider produit. **Proposition** : red-agent insère placeholders ≤ 28 chars (e.g. valeurs EN copiées), tracking ticket pour traduction produit post-green.
- **Q4 (`Animated` API choice — `Animated` RN vs Reanimated)** : Pour la transition expanded ↔ mini-collapsed, on peut utiliser `Animated` RN (simple, pas de worklet) ou `Reanimated` (cohérent avec pinch). **Proposition** : `Animated` RN pour la transition collapse (simple `Animated.timing` 200ms), `Reanimated` pour pinch (obligatoire pour worklet-driven `useSharedValue`). Hybride accepté car les deux libs sont déjà imports usage existant (`ImageFullscreenModal` = RN, walk-mode = Reanimated).
- **Q5 (Telemetry sentry breadcrumb ?)** : R28 demande `console.debug` uniquement. La forme breadcrumb Sentry n'est-elle pas déjà conventionnée ailleurs ? **Proposition** : `console.debug` only en V1 (zero PII, zero new infra). Sentry breadcrumb deferred si signal utile post-bake.
