# `security@musaium.com` mailbox setup

**Audience:** Musaium founder (Tech Lead).
**Goal:** Bring the mailbox advertised in [`SECURITY.md`](../../SECURITY.md) into existence so the VDP policy is enforceable, not aspirational. Without a real, monitored mailbox, the policy is a misrepresentation under GDPR Art. 32 (and a CRA Art. 14 trail problem if a CVE lands).
**Last updated:** 2026-05-17. **Audit run:** `2026-05-17-w4-compliance-ops-release`.
**TL executes** — this runbook is deterministic, the actions are human-loop (DNS edits, SaaS console clicks, sending a real test email).

## 1. Provider

| Plan | Storage | Price (€/mo HT) | Why |
|---|---|---|---|
| OVH Pro Mail "MX Plan 25 GB" | 25 GB | 1.99 / address | Single mailbox is enough — no shared mailbox needed pre-V1. Upgrade to MX Pro post-launch if volume grows. |

## 2. DNS records

Edit at **OVH Manager → Domains → `musaium.com` → DNS Zone**. The apex (`musaium.com`) MX must point to OVH MX servers (`mx0.mail.ovh.net.` / `mx1.mail.ovh.net.`) — verify the existing zone does not collide with an old Gandi / Cloudflare MX.

```
; ── MX ──────────────────────────────────────────
musaium.com.    3600    IN  MX  1   mx0.mail.ovh.net.
musaium.com.    3600    IN  MX  5   mx1.mail.ovh.net.
musaium.com.    3600    IN  MX  10  mx2.mail.ovh.net.

; ── SPF (anti-spoofing) ────────────────────────
musaium.com.    3600    IN  TXT "v=spf1 include:mx.ovh.com -all"

; ── DKIM (generated by OVH; copy from console) ─
ovh1._domainkey.musaium.com. 3600 IN TXT "v=DKIM1; k=rsa; p=<COPY FROM OVH CONSOLE>"

; ── DMARC (reject) ─────────────────────────────
_dmarc.musaium.com.  3600 IN TXT "v=DMARC1; p=quarantine; rua=mailto:security@musaium.com; ruf=mailto:security@musaium.com; pct=100; aspf=s; adkim=s"
```

DMARC starts at `p=quarantine`. After 14 days of clean reports (no legit traffic in `rua`), switch to `p=reject`. Do NOT skip the quarantine phase — a misconfigured DKIM will silently nuke transactional mail from `noreply@musaium.com`.

DO NOT set an auto-reply on `security@`. ## 6. Verification — end-to-end smoke

Run these in order. Each must produce the documented evidence for the audit-360 W4 PR.

### 6.1 DNS propagation (wait 1 h after edits)

```bash
dig +short MX musaium.com
# Expected: 1 mx0.mail.ovh.net.  5 mx1.mail.ovh.net.  10 mx2.mail.ovh.net.

dig +short TXT musaium.com | grep -i spf
# Expected: "v=spf1 include:mx.ovh.com -all"

dig +short TXT _dmarc.musaium.com
# Expected: "v=DMARC1; p=quarantine; rua=mailto:security@musaium.com; ..."

dig +short TXT ovh1._domainkey.musaium.com
# Expected: "v=DKIM1; k=rsa; p=..."
```

Screenshot the four outputs → `team-state/2026-05-17-w4-compliance-ops-release/evidence/c8.1-dns.png` (or paste as text in the PR description).

### 6.2 External delivery test

From a non-Musaium domain (gmail/outlook):

```
To: security@musaium.com
Subject: [TEST 2026-05-17] VDP mailbox smoke
Body: Ignoring this is fine; just verifying inbound delivery + forwards. Reply-to: <sender>.
```

Verify within 5 minutes:

1. Email landed in `security@musaium.com` inbox (login via OVH webmail).
2. Email landed in founder's personal inbox (forward worked).
3. Email landed in Slack `#security` channel (forward worked).
4. Open the header in OVH webmail and confirm `Authentication-Results: ... spf=pass ... dkim=pass ... dmarc=pass` for inbound. *Inbound* DMARC pass is informational; the SPF/DKIM-pass-on-our-domain matters for *outbound* (next step).

### 6.3 Outbound test (signs SPF+DKIM correctly)

From OVH webmail, send to `check-auth@verifier.port25.com` (Port25 free DKIM/SPF/DMARC verifier). The bounce-back report must show:

```
==========================================================
Summary of Results
==========================================================
SPF check:          pass
"iprev" check:      pass
DKIM check:         pass
DKIM check:         pass (DKIM signature on subject)
DMARC check:        pass
==========================================================
```

Save the verifier bounce as `team-state/2026-05-17-w4-compliance-ops-release/evidence/c8.1-port25.eml`.

### 6.4 Wrong-domain test (spoof rejection)

From a Gmail account, change the From: header to `security@musaium.com` (use Gmail "Send mail as" → manually set). It MUST bounce / land in spam at any receiving Gmail account. This validates DMARC `p=quarantine` is enforced.
