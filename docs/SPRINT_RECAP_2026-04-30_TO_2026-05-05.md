### F7 — httpOnly cookies + CSRF double-submit (BE + web)

**What:** Backend issue `access_token` + `refresh_token` (les deux httpOnly + Secure + SameSite=Strict) + `csrf_token` (NOT httpOnly pour que JS puisse l'écho) sur login/refresh/social-login. CSRF middleware valide:

1. Double-submit (header `X-CSRF-Token` MUST match cookie)
2. **HMAC binding** : `csrf_token = HMAC-SHA256(access_token, CSRF_SECRET)` — cookie MUST = HMAC du access token, pas n'importe quelle valeur attaquant écho both places
3. Constant-time compare via `crypto.timingSafeEqual` sur length-padded buffers

**Mobile keeps reading JSON envelope** : skip rules sur Authorization headers (browsers ne les auto-send pas).

**Where:**

- BE commit `1cfb61d7e`. Cookie helper `museum-backend/src/modules/auth/adapters/primary/http/helpers/auth-cookies.ts:45-97`. CSRF middleware `museum-backend/src/helpers/middleware/csrf.middleware.ts:77-114`.
- Web commit `814fbc193`. CSRF echo `museum-web/src/lib/api.ts:64-67,107-111,169-176`.

**Code excerpt (`csrf.middleware.ts:96-111`)**

```ts
const headerToken = req.headers['x-csrf-token'];
```

### F-DiD — Per-attempted-account login limiter (CGNAT bypass)

**What:** layered limiter — coarse IP (10 req / 5 min) **PLUS** per-account bucket keyed `email:<lowercased+trimmed>` (20 req / 5 min). Returns 429 **AVANT** password compare (no enumeration oracle via timing).

**Where:** commit `d3be21672` + `museum-backend/src/modules/auth/adapters/primary/http/helpers/auth-rate-limiters.ts:32-42`.

**Pourquoi:** sans ça, attaquant derrière CGNAT (un IP partagé par 100s users) ne peut pas être rate-limité — credential-stuffing ciblé sur un compte spécifique passait à travers l'IP-only bucket.

```ts
export const loginByAccountLimiter: RequestHandler = createRateLimitMiddleware({
  limit: toPositiveInt(process.env.AUTH_LOGIN_ACCOUNT_RATE_LIMIT, 20),
  windowMs: toPositiveInt(process.env.AUTH_LOGIN_ACCOUNT_RATE_WINDOW_MS, 5 * 60_000),
  keyGenerator: (req) => {
    const email = (req.body as { email?: unknown } | undefined)?.email;

      ? `email:${email.trim().toLowerCase()}`
```

### LLM judge budget Redis SET+TTL multi-instance (ADR-030 + ping-gate fix `d7ba7f5f7`)

**Pre-fix problem:** F4 LLM-judge counter était per-process — 5€/jour cap effectivement = 5€ × replicas/jour.

**Post-fix:** shared Redis key `guardrail:judge:budget:<utcDayKey>` via `cache.incrBy(key, cents, ttl)` Lua atomic INCRBY+EXPIRE. Lazy daily reset + 60s buffer past midnight.

**Critical review fix `d7ba7f5f7`:** ADR-030 Redis adapter pre-fix returned 0 cents on `cache.get(...) === null` — c'est **fail-OPEN** (Redis outage → null → "0 cents spent" → judge runs uncapped). **Le "Redis-DDoS bypass" attack était real** (kill Redis, watch judge run uncapped).

**Fix:** `cumulativeCents()` calls `cache.ping()` first ; if ping false / throws → return `Number.POSITIVE_INFINITY` (fail-CLOSED, judge skipped) ; if ping true + null → legitimate first-of-day miss = 0 ; if ping true + counter NaN/negative → Infinity. Caught dans code review (reviewer fresh-context, weightedMean 89.05).

```ts
async cumulativeCents(): Promise<number> {
  // Reachability gate — closes the outage-vs-miss ambiguity.
  const reachable = await this.cache.ping().catch(() => false);
  if (!reachable) {
    logger.warn('guardrail_judge_budget_redis_unreachable_fail_closed', {});
    return Number.POSITIVE_INFINITY;
  }
  const value = await this.cache.get<number>(this.keyForToday());
  if (value === null) return 0;
  if (!Number.isFinite(value) || value < 0) {
    logger.warn('guardrail_judge_budget_counter_invalid', { value });
    return Number.POSITIVE_INFINITY;
  }
  return value;
}
```

**Code excerpt (`llm-cache.service.ts:32-53`)**

```ts
classify(input: LlmCacheKeyInput): LlmContextClass {
  if (input.userPreferencesHash) {
    return 'personalized';
  }
  if (input.museumContext?.museumId !== undefined && input.museumContext.museumId !== null) {
    return 'museum-mode';
  }
  return 'generic';
}

async lookup<T>(input: LlmCacheKeyInput): Promise<LlmCacheLookupResult<T>> {
  const contextClass = this.classify(input);
  const key = this.buildKey(input, contextClass);
  const value = await this.cache.get<T>(key);
  if (value !== null) {
    llmCacheHitsTotal.inc({ context_class: contextClass });
  } else {
    llmCacheMissesTotal.inc({ context_class: contextClass });
  }
  return { hit: value !== null, value, contextClass };
}
```

**Why this matters:** **première surface cost-control sur LLM spend**. Per-user scope baked into key prevents cross-user leakage. Adaptive TTL aligns avec cost model.
