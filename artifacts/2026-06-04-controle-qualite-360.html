Résumé exécutif

Au niveau du code applicatif, Musaium est un repo solo-dev pré-launch genuinement mûr. Le défaut n'est presque jamais l'exécution — ce sont les garde-fous censés garantir la qualité dans le temps, et le timing de l'ingénierie.

Ce qui est vraiment solide

  • Sécurité : auth JWT à familles, rotation refresh banking-grade, BOLA fermé, secrets prod fail-fast, CSRF/CSP/SSRF.
  • Assainissement IA réel à 6 couches câblées (egress LLM + double-scrub Sentry/Langfuse, fuite GPS RGPD Art.7 fermée).
  • Conformité 2026 : AI Act Art.50 (1ʳᵉ interaction), RGPD Art.17 erasure multi-store, plancher CNIL 15 ans.
  • Discipline type/lint : zéro as any en prod sur ~150k LOC.
  • Densité rare de leçons réellement enforcées par outillage (lint / ast-grep / sentinels).

Les 4 vérités systémiques re-vérifiées

  • Garde-fous désarmés silencieusement : Stryker if:false + boundaries no-op prouvé + fail-CLOSED V2 en continue-on-error.
  • Bug d'intégrité dans une lib de sécurité : computeRowHash droppe le nested (collision reproduite) — chemin notification CNIL.
  • Couche sécurité IA non gardée en CI : la garantie fail-closed (ADR-047) n'est validée par aucun gate bloquant.
  • Sur-ingénierie de timing : primitives 100k MAU posées sur un produit B2C à 0 utilisateur.

Bonne nouvelle re-vérifiée : 2 des 3 piliers que la cartographie-360 (2026-05-31) disait « désarmés » sont réellement remédiés — le faux-vert e2e Maestro est fermé par double-gate, et le frozen-test est un vrai hook (plus pur honor-system, mais limité à /team). La mémoire ne mentait pas là-dessus ; elle se trompe sur le 3ᵉ : Stryker reste off.

Findings HIGH — confirmés, re-vérifiés

CRITICAL? AUDIT-01 — audit-chain.computeRowHash droppe le contenu imbriqué du hash

audit-chain.ts:43-46 appelle JSON.stringify(metadata, Object.keys(metadata).sort()). Le 2ᵉ argument est un replacer-allowlist appliqué récursivement : Object.keys ne liste que le premier niveau, donc tout objet imbriqué (breach:{…}, provider:{…}) est sérialisé sans ses sous-clés → {"breach":{}}. Deux payloads forensiques différents produisent le même hash. Collision reproduite empiriquement (COLLISION=true) + re-lue à la main par l'orchestrateur.

Impact : la chaîne d'audit « inviolable » ne couvre pas le contenu nested. Atteint le chemin de notification de violation CNIL (guardrail-audit-payload.ts:105, audit.service.ts:207-219) où tout le payload est imbriqué → légalement opposable. Invisible aux tests, qui hardcodent le sérialiseur buggé comme oracle (AUDIT-02).

Fix : sérialiseur canonique deep-recursif (clés triées récursivement) + re-stamp/version des hashes historiques + tests nested sur l'unit ET la parité migration. → Action #1, via /team.

HIGH ARCH-01 — l'enforcement hexagonal backend (eslint-plugin-boundaries) est un no-op prouvé

eslint.config.mjs:64-160 : le bloc boundaries n'a aucun import/resolver dans son settings (seul un import-x/resolver vit dans un bloc séparé). Sans résolveur, les imports @modules/* résolvent en external (path:null) → la règle ne classe rien → ne fire jamais. Reproduit : un import domain→infrastructure flagrant = 0 erreur ; ajouter import/resolver.typescript au bloc fait fire la règle. Re-lu à la main par l'orchestrateur.

Mensonge de config : le commentaire eslint.config.mjs:115-118 affirme que « la migration v6 a restauré l'enforcement » — faux. Et une vraie fuite existe déjà : chat/domain/ports/chat-orchestrator.port.ts:9 importe un type défini en useCase (ARCH-02).

Fix : ajouter le resolver au bloc boundaries + fixture-garde CI + un sentinel fs-based indépendant (modèle FE). → Actions #2 & #6.

HIGH TQ-01 — la garantie fail-CLOSED V2 (ADR-047) n'est gardée par aucun gate

ci-cd-backend.yml:524-565 : le job ai-tests est en continue-on-error:true (l.564) et sans sidecar → guardrail-v2-live throw en beforeAll et l'erreur est avalée. Les invariants déterministes (dead-port/dead-URL/budget/fail-soft) sont co-localisés avec des asserts LLM non-déterministes → ils tombent ensemble. ci-cd-llm-guard.yml ne fait que build+health-smoke.

Fix : sortir les tests fail-CLOSED déterministes du describe live-sidecar → gate bloquant sans sidecar ; garder les asserts LLM en advisory. → Action #3.

HIGH PILLAR-01 / ARCH-04 — le mutation gate Stryker est if: false

ci-cd-backend.yml:411 : le job entier est désactivé depuis 2026-05-09 ; mutation/mutation:ci/mutation:gate ne tournent jamais (même le cron nightly est skippé). La doctrine « kill ≥80 % = signal load-bearing » (jest.config.ts:126) est adossée à zéro enforcement. Honnête dans le code (commentaire CI : « ne pas citer comme garde actif »).

Fix : trancher — re-armer (plan ci-cd-backend.yml:405-410) OU acter formellement que la force des tests n'est mesurée que par couverture, jamais citée comme garde. → Action #4. (Le vérificateur ramène ARCH-04 à MEDIUM car désactivation intentionnelle/documentée.)

HIGH AISAN-01 — 64 chars de texte utilisateur brut écrits dans l'audit, avant le sanitizer PII

guardrail-snippet.ts:36-40 : pour tout input bloqué, le snippet garde 64 caractères de texte utilisateur brut (email/téléphone possibles) dans la chaîne de hash d'audit (rétention 13 mois), avant le passage par RegexPiiSanitizer — alors que l'entrée REDACTED, elle, est bien sanitisée. Contredit le commentaire LLM02 du fichier.

Fix : passer fullText au sanitizer PII avant slice(0,64) ; garder un fingerprint sha256 du texte intégral pour le dédup. → Action #5.

HIGH SYS-02 — soft-delete deletedAt non filtré hors login (email-squat + tokens à comptes supprimés)

Le filtre deletedAt n'est appliqué qu'au login : forgotPassword émet des tokens de reset à des comptes supprimés, et registerUser/changeEmail laissent un compte soft-deleted squatter l'email (unicité non exclue des soft-deleted). Fuite déjà active sur 3 callsites, pas future.

Fix : filtrer deletedAt dans forgotPassword + exclure les soft-deleted des checks d'unicité email (ou migrer vers @DeleteDateColumn). → Action #7 (avec SYS-01).

Dettes systémiques transverses (« une dette trouvée quelque part… est-elle partout ? »)

C'est la question que tu as explicitement posée. Réponse : oui pour deux classes, et les sentinels figés en laissent fuiter une troisième.
