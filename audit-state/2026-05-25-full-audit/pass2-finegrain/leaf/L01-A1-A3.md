## P0.A3 — Sentry tag URL leak + event.tags scrubbed in beforeSend

**VERDICT : DONE** (matches roadmap `✅`). Implementation is **broader than the literal left-column statement** — see divergence.

**SENSITIVE_QUERY_KEYS** — `packages/musaium-shared/src/observability/sentry-scrubber.ts:29-41` : 11-entry Set = `access_token, api_key, apikey, code, email, password, phone, refresh_token, secret, state, token`. **scrubUrl** — shared `:142-157` : splits on first `?`, lower-cases key, redacts matched keys.

**event.tags scrubbed in scrubEvent** — shared `:220-232` : walks `event.tags`, applies `scrubRecord` (body-field keys), then `SENSITIVE_HEADER_REGEX` keys → REDACTED, then `scrubUrl` on `isUrlLikeValue` values. `ScrubbableEvent.tags` field added `:71`.

**Second (source) chokepoint** — `sentry.ts:109-110` `captureExceptionWithContext` : `const scrubbed = isUrlLikeValue(value) ? scrubUrl(value) : value; scope.setTag(key, scrubbed)`. This scrubs the `path: req.originalUrl` tag passed from `error.middleware.ts:97` BEFORE Sentry indexes it. Defense-in-depth with beforeSend.

**Test coverage (shared pkg, comprehensive)** — `packages/musaium-shared/src/observability/sentry-scrubber.test.ts`:

**Parity sentinel (genuinely wired, not orphaned)** — `scripts/sentinels/sentry-scrubber-parity.mjs` (repo root, NOT museum-backend/scripts) : `CANONICAL_HASH` of the canonical scrubber + 3 wrappers, documents 7→11 extension (`:66`). Wired: `.husky/pre-push:95`, `.github/workflows/sentinel-mirror.yml:120`, `package.json:20` `sentinel:sentry-scrubber-parity`.
