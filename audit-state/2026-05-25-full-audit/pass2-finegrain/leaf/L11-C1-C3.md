### CC-BY-SA — statut : **INATTEIGNABLE (bug confirmé)**
Chaîne tracée from scratch :
1. CLI accepte `--license-filter=cc-by-sa` (`catalog-ingest.ts:435`) → filtre = `['cc-by-sa']`.
2. Œuvre Wikidata CC-BY-SA-4.0 → `?license` = `http://www.wikidata.org/entity/Q18199165`.
3. `mapLicenseUriToSlug` → **`cc-by-sa-4.0`** (`:58`).
4. `classifyLicense('cc-by-sa-4.0', ['cc-by-sa'])` → `['cc-by-sa'].includes('cc-by-sa-4.0')` = **false** → `null` → REJETÉ.
   - Aucune normalisation `cc-by-sa-4.0` → `cc-by-sa`. Le slug produit ne matche AUCUN `AllowedLicense`.
5. Défense secondaire DB : `1778406339944-AddArtworkEmbeddings.ts:60-61` `CHECK ("license" IN ('public-domain', 'cc-0'))` — n'inclut **même pas** `'cc-by-sa'`. Donc même si (4) passait, l'INSERT serait rejeté par contrainte SQL.

→ **CC-BY-SA est doublement inatteignable** : (a) mismatch slug `cc-by-sa-4.0` vs `cc-by-sa`, (b) DB CHECK 2-valeurs. Idem `cc-by`, `cc-by-4.0`, `gfdl-1.2` (mappés « forward-compat » mais 100 % rejetés — cf. docstring `:49-51` qui l'admet).

**Incohérence à 3 couches** (vérifiée) :
- DB CHECK : `{public-domain, cc-0}` (`AddArtworkEmbeddings.ts:61`)
- `AllowedLicense` (filtre CLI) : `{public-domain, cc-0, cc-by-sa}` (`catalog-ingest.ts:67`)
- `ArtworkImageLicense` (entité) : `{public-domain, cc-0, cc-by-sa, unknown}` (`artworkEmbedding.entity.ts:22`)
- slugs produits par la table : `{..., cc-by-sa-4.0, cc-by, ...}`

Le CLI **annonce** `cc-by-sa` comme valeur de filtre valide alors qu'elle produit systématiquement zéro insertion (rejet silencieux compté en `licenseRejected`). C'est exactement la classe de bug que C2 prétend corriger, déplacée de pd/cc-0 vers cc-by-sa.

### Fixture de test — MASQUAGE PARTIEL (la question du brief)
- **Unit `catalog-ingest.helpers.test.ts`** : CORRIGÉ — injecte les **vraies URIs** (`WIKIDATA_URI_PUBLIC_DOMAIN = Q19652`, `:42`, `:131`, `:139`, `:191`). Teste `mapLicenseUriToSlug` pour public-domain (`:347`), cc-0 (`:351`), unknown→null (`:355`), vide→null (`:362`). **MAIS aucun test CC-BY-SA** : ni Q18199165, ni assertion que le slug produit est atteignable par l'allow-list. Le bug cc-by-sa est hors couverture.
- **Intégration `catalog-ingest.test.ts`** : **MASQUE TOUJOURS** le chemin URI. `makeBinding` (`:97-110`) injecte `license: { value: license }` où `license` est un **slug** (`'public-domain'|'cc-0'|'cc-by-sa'`, `:100`, `:108`). Le test passe par le vrai `defaultFetchArtworksOfMuseum` (pas d'override `fetchArtworks`), donc `mapLicenseUriToSlug('public-domain')` hit le branche pass-through → slug inchangé → matche allow-list. **Le test ne fait JAMAIS transiter une URI Wikidata.** Le test cc-by-sa (`:235-259`) injecte le slug `'cc-by-sa'` (pas Q18199165) et asserte rejet (car filtre pd+cc-0) — ne prouve RIEN sur le mapping URI ni sur l'inatteignabilité.
- **`catalog-ingest.museum-id.test.ts:140`** : idem, slug `'public-domain'` (le commentaire `:120-128` admet explicitement « this fixture uses the slug … NOT a URI »).

→ La correction de fixture promise par l'énoncé (« le fixture injecte le slug → bug invisible ») a été faite **au niveau unit pour pd/cc-0 seulement**. L'intégration end-to-end injecte encore des slugs. **Le bug CC-BY-SA reste invisible aux tests.**

**Conclusion C2** : le fix pd/cc-0 est réel et testé (unit). MAIS (1) CC-BY-SA est inatteignable par mismatch de slug `cc-by-sa-4.0` vs `cc-by-sa` non normalisé, aggravé par un DB CHECK 2-valeurs ; (2) le fixture d'intégration masque toujours le chemin URI ; (3) aucun test ne couvre CC-BY-SA. **Dette : normaliser `cc-by-sa-4.0`→`cc-by-sa` (ou retirer `cc-by-sa` de `AllowedLicense` + élargir DB CHECK + déclarer V1 = pd/cc-0 strict, cohérent avec spec §8 Q2), ajouter test URI CC-BY-SA, migrer fixture intégration vers URIs.** Note : si l'intention V1 est pd/cc-0 strict (spec §8 Q2 « CC-BY-SA rejected »), alors le bug est d'**exposer** `cc-by-sa` comme filtre CLI accepté — fausse promesse.
